```
Schema --compile--> Grammar
                      |
请求 --> Matcher 合法集 --> bitmask --> logits' --> sample --> token
             ^                                         |
             +-------------- 更新合法集 -----------------+
```

口播要点：主模型算子不变；插入点在 logits 后；异步时 bitmask 必须对齐当前步。

## 4. 白板②：KV 亲和因果

```
高前缀重复
    |
tokenize@Coordinator (6ms)
    |
token 最长前缀 × 全局索引
    |
命中 --> 少跑 Prefill(PFA+Linear*) --> TTFT↓ (案例 -70%)
未命中 --> 满量 Prefill
    |
同时 load_gated --> 保护 Decode(IFA+FFN GEMV) 不被挤爆
```

口播要点：数字是场景验证；机制是少跑计算密集段；不是 kernel 加速 70%。

## 5. 白板③：优化分层（防被问懵）

```
L5 业务路由/亲和/PD          ← 你的主交付
L4 连续批处理/异步引擎        ← 错位 bug 所在
L3 图模式 GE/aclgraph         ← 懂选型
L2 融合算子 / FA / 量化       ← 读过源码与文档
L1 硬件 Cube/Vector/HBM       ← Roofline 语言
```

口播：我主要在 L5/L4 交付，L3–L1 用于正确决策与协作。

## 6. 压力问题标准答（背过）

**「你这不就是调包吗？」**  
> 结构化输出从 0 到 1 含编译缓存（SHA256+FIFO≈100）、后端抽象、并发正确性，五千行+；亲和是多实例调度策略与客户验证。调的是引擎与索引，不是调一个 YAML。算子库我调用并理解其代价模型。

**「昇腾算子你能讲多深？」**  
> 能讲 PFA/IFA 切分差异、online softmax、GE vs aclgraph、MoE/GMM/MC2 动机；对过 FIA online softmax 源码。不能假装写过生产 AscendC。

**「下一个技术成长？」**  
> 在现有框架阵地上加深：亲和×PD 的 KV 传递观测、约束解码与 Graph 分段、与算子团队共建热点融合（如 mask+sample）。可补一个小范围 AscendC 实验，但不替代框架主线。

## 7. 模拟追问快问快答（10 连）

1. Prefill 为何贵？→ OI∝S_q，计算密集。  
2. Decode 为何访存？→ M≈1，搬 W/KV。  
3. 亲和动哪个指标？→ 主要 TTFT。  
4. bitmask 哪个单元？→ Vector。  
5. 假命中后果？→ 错 KV 或退回满 Prefill。  
6. aclgraph 融不融合？→ 不融合。  
7. PFA 切谁？→ 切 Q。  
8. IFA 为何 reduce？→ 切同一行的 KV。  
9. load_gated 防什么？→ Decode 过载。  
10. 你的边界？→ 框架交付；算子选型与归因。

**加问（交界补强）**  
11. 传 KV 还是重算？→ 比 T_xfer 与 T_prefill（[`16`](./16)）。  
12. mask 与 top-p 顺序？→ 先 mask（[`15`](./15)）。  
13. MLA Decode 一律算力密集？→ 否，看 S_kv/MTP（[`19`](./19)）。  
14. 编译缓存淘汰策略？→ FIFO≈100，勿说 LRU（[`24`](./24)）。  
15. Tool vs 结构化输出？→ 解析 vs 生成期约束（[`23`](./23)）。
