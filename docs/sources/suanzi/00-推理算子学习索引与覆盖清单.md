## 2. Decode Step 算子地图（把零散知识钉在时间线上）

```
Embed → RMSNorm → QKV proj → RoPE → KV write → Attention → O proj
      → RMSNorm → FFN/MoE → … ×L 层 → LM Head → Sampler
```

| Step | 阶段 | 算子层在干什么 | 本仓主落点 | 常见瓶颈 |
|:---:|------|----------------|------------|----------|
| 0 | Embed | 查表 | [`22`](./22-Embed半页.md) | 通常小 |
| 1 | Pre-Attn Norm | Vector 归一化 | [`02`](./02-Norm-RoPE-融合.md)；常融入 `qkv_rms_norm_rope_cache` | 单独做则 Host/小算子开销大 |
| 2 | QKV proj | Cube MatMul/GEMV | [`01`](./01-Linear-FFN-MatMul-SwiGLU.md)；MLA：`mla_preprocess/prolog`；TP：`mc2/*` | Decode 访存密集 |
| 3 | RoPE | Vector 旋转 | [`02`](./02-Norm-RoPE-融合.md)；`apply_rotary_pos_emb` / 融入 1–2 | 小算子，宜融合 |
| 4 | KV write | 写 Paged/连续 cache | [`03`](./03-Attention家族-Paged-MLA.md)；`scatter_pa_kv_cache`、`rope_quant_kvcache` | 带宽 |
| 5 | Attention | FA 分块 + online softmax | [`03`](./03)；Prefill:`PFA` Decode:`IFA` 统一:`FIA` | Prefill 算力 / Decode 带宽 |
| 6 | O proj | MatMul + 常 AllReduce | [`01`](./01)；`matmul_all_reduce`（[`04`](./04) MC2） | TP 通信 |
| 7 | Post Norm | Vector | [`02`](./02)；常融入 `*_add_rms_norm` | 同 1 |
| 8a | Dense FFN | GateUp+SwiGLU+Down | [`01`](./01)；`ffn/ffn`、`grouped_matmul_swiglu_quant` | Decode 访存；大 batch 可翻计算 |
| 8b | MoE | gating→permute→GMM→finalize | [`04`](./04-MoE与通算融合MC2.md) | EP 通信 + 专家负载 |
| 9 | LM Head | MatMul（N=V） | [`18`](./18-LMHead与Vocab并行.md) | 大 V 带宽；vocab parallel |
| 10 | Sampler (+mask) | Vector / 图外 | [`15`](./15)、[`11`](./11)、[`23`](./23-ToolCall与结构化输出交界.md) | Host/错位/约束 |

| 编译缓存 SHA256+FIFO（容量约 100，命中不调序） | Host 层，非 Device FLOPs | `09` `13` `15` |
| mask/采样错位 | 异步 CB 时序对齐 | `10` `11` `15` |
| Tool Call / DSV3 | 解析交付；模型侧转 MLA/MoE 知识点 | `04` `09` `19` |
| Server 重构 | 可维护性，便于接 Graph/采样钩子 | `09` |
