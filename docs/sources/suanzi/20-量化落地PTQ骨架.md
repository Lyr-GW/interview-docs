# 20 · 量化落地骨架（PTQ / 敏感层 / 验收）

> [`05`](./05-量化与KVCache算子.md) 讲清「为什么量化、W/A/KV 粒度」。  
> 本篇补面试常卡的：**怎么开、怎么验收、坏了怎么回退**——骨架级，非完整校准工程手册。  
> 简历：未列个人量化交付；用于防守「你们线上怎么开量化」类追问。

---

## L0 · 三条路径别混

| 路径 | 动谁 | 主要省什么 | 典型风险 |
|------|------|------------|----------|
| **W8A8 / W4A** | 权重（+激活） | 权重带宽、显存；Decode Linear/FFN | 敏感层精度崩 |
| **KV 量化** | Cache | KV 带宽与显存；IFA 读路径 | 长上下文累积误差 |
| **权重量化 + KV 量化** | 两者 | 叠加 | 调试归因更难，建议分步开 |

**金句**：量化是 **memory-bound 阶段的主武器**；对已 compute-bound 的 Prefill 大块，收益相对小（仍可能省显存、提 batch）。

---

## L1 · PTQ 流程骨架（口述用）

```
1. 选精度目标：W8 / W4 / KV int8/fp8 …
2. 校准集：代表性 prompt（含长上下文、工具调用、结构化输出若业务相关）
3. 标定 scale：per-tensor / per-channel / per-token（激活常 per-token）
4. 敏感层策略：首尾层、部分 Attention out_proj、LM Head 等可留高精或跳过
5. 导出 / 融合：与 RoPE+Quant+写 cache、W8A8 GEMM 路径对齐（见 ops-transformer 相关融合）
6. 验收：任务指标 + 服务指标（见下）
7. 回退开关：按层 / 按模块一键回 FP16/BF16
```

### 敏感层（经验向，非教条）

- **Embedding / LM Head**：词表投影，常对量化敏感 → 优先保留高精或单独策略（接 [`18`](./18-LMHead与Vocab并行.md)）。  
- **部分 Norm 后尺度**：极端 scale 易炸。  
- **MoE 路由 logits**：量化可能改变专家选择 → 负载与精度双风险（[`04`](./04-MoE与通算融合MC2.md)）。

面试说「我们会做敏感层分析与跳过」即可，**勿编造你未跑过的具体层号**。

## L1 · 验收指标（两本账）

| 账本 | 看什么 | 失败信号 |
|------|--------|----------|
| **质量** | 业务集准确率 / perplexity / JSON 合法率（结构化输出） | 掉点超 SLO；非法 JSON↑ |
| **服务** | TTFT / TPOT / 吞吐 / 显存 / batch | 精度换来的带宽收益被 Host/通信吃掉 |

结构化输出场景加测：

- 开 bitmask 时，量化后 **合法集约束仍满足**；  
- 勿只看开放生成 BLEU。

## L2 · 与简历特性的交界

| 特性 | 量化交界 |
|------|----------|
| KV 亲和 / PD 传 KV | KV 量化 → [`16`](./16) 传输体积↓，B 路径更香 |
| 结构化输出 | 质量验收加 Schema 合法率；bitmask 不替代量化 |
| load_gated | 量化抬 Decode batch 上限时，更要防过热实例 |

**禁止说法**：把 TTFT −70% 说成「我做了 W8A8」。更多口误对照：[`24`](./24-答辩用词对照.md)。

---

## L2 · 算子脚印（昇腾对照一句）

- W 量化 GEMM / `grouped_matmul_*quant`；  
- `rope_quant_kvcache` 类：RoPE + 量化 + 写 cache 融合减往返；  
- Decode IFA 读量化 KV 时 dequant 时机见 [`05`](./05-量化与KVCache算子.md)。

---

## L3 · 30 秒口述

> 量化先分清动权重还是动 KV。PTQ 是校准→敏感层→导出→双账本验收→可回退。Decode/访存密集收益最大；Prefill 算力密集时主要赚显存与可服务 batch。我个人交付在调度与约束解码，量化按这个骨架理解与协作，不声称独立交付校准流水线。
