# 专题 07：拓展阅读 / 延伸知识（适度扩展）

> 面试官在反问环节透露：他们靠"配置方式"做加速（PD 分离等），用 SGLang，期望候选人能做框架级修改、算子优化、引入新投机采样方法。以下方向按"下次面试可能被问 + 对目标岗位有用"排序。

---

## 1. PD 分离的深入方向

- **开山与演进**：DistServe（OSDI'24，goodput 驱动的 P/D 资源配比与放置）→ Mooncake（FAST'25，KVCache 中心 + 过载调度）→ DeepSeek-V3/R1 线上大 EP + PD 分离实践 → vLLM/SGLang 原生 PD 支持（NIXL、Mooncake、LMCache connector）。
- **关键工程问题**：KV 传输与 prefill 计算的 overlap（逐层/分块流式传输）、P:D 配比的动态调整（潮汐流量下 P↔D 角色互换，Motor 的 `scale_p2d.py` 即此类）、传输层选型（RDMA/NVLink/NIXL）。
- **何时不该用**：短序列、低并发、单机场景，传输开销与运维复杂度盖过收益。

## 2. KV cache 压缩 / 量化 / 卸载

- **量化**：KV cache FP8（vLLM `--kv-cache-dtype fp8`，`vllm/config/cache.py` 的 `cache_dtype`）近乎免费地翻倍 KV 容量；更激进的 2/4-bit KV 量化（KIVI、KVQuant）。
- **驱逐/稀疏化**：H2O（heavy-hitter oracle，按注意力分数保留重要 token）、SnapKV、StreamingLLM（attention sink + 滑窗）；长上下文下减 KV 占用与访存。
- **结构性压缩**：MLA（DeepSeek-V2/V3 的 multi-head latent attention，把 KV 压成低秩 latent，从模型结构上砍 KV）、GQA/MQA；跨层共享（YOCO、CLA）。
- **分层存储**：GPU → CPU DRAM → SSD 的层级 KV（Mooncake Store、SGLang HiCache L1/L2/L3、LMCache），"KV cache 即存储系统"是 2025–2026 的明确趋势。

## 3. 注意力与算子层

- FlashAttention-3（Hopper 上 warp 专用化/异步流水）、FlashInfer（推理专用 attention 库，vLLM/SGLang 均集成）；
- MoE 推理：expert parallel 的通信优化（DeepEP、all-to-all overlap）、专家负载均衡；
- 昇腾背景的迁移价值：ATB 算子经验（工作区 `MindIE-LLM/examples/atb_models/` 含 469 个 C/C++ 与 1019 个 Python 文件）可类比 CUDA kernel 优化叙事。

## 4. 长上下文推理

- 上下文并行（context parallel / ring attention）；
- prefill 加速：chunked pipeline（Mooncake 的跨节点流水 prefill）、稀疏 attention prefill（MInference、NSA/DSA——DeepSeek 稀疏注意力路线）。

## 5. 调度与服务层前沿

- **抢占与优先级**：vLLM V1 的 priority scheduling、SLO 感知调度、prediction-based early rejection（Mooncake）；
- **Continuous batching 之后**：async scheduling（CPU/GPU overlap）、multi-step scheduling；
- **多模型服务**：semantic routing / 强弱模型级联（专题 06）、serverless LLM（模型快速冷启动）；
- **Agent/多轮时代的缓存**：tool call 循环中 KV 复用（前缀树跨请求共享）、prompt cache 计费（各大 API 已产品化）。
