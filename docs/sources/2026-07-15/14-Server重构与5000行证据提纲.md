## 2 · Server / C++ 重构目录（已定位）

### 2.1 架构落点（有路径可指）

在 `MindIE-LLM-1` / `MindIE-LLM-PyServer` / `MindIE-LLM_pub`：

```text
src/server/endpoint/
├── single_llm_req_handler/          # Handler：Base → Prefill / Decode / PnD
│   ├── single_llm_req_handler_base.{h,cpp}
│   ├── single_llm_prefill_req_handler.{h,cpp}
│   ├── single_llm_decode_req_handler.{h,cpp}
│   └── single_llm_pnd_req_handler.{h,cpp}

│   ├── single_req_infer_interface_base.{h,cpp}
│   ├── parse_protocol.{h,cpp}
│   └── single_req_*_infer_interface.{h,cpp}
└── utils/infer_param.{h,cpp}        # 含 MTP×structured 硬互斥
```

| 项 | 核实 |
|----|------|
| `enable_shared_from_this` + make_shared 注释 | `single_req_infer_interface_base.h` 约 L43–46 |
| Handler 现状合计 | `wc` ≈ **2320** 行（`.h+.cpp`）——**现状快照，不是删行前数字** |
| Interface 现状合计 | `wc` ≈ **7239** 行——同上，**禁止**说成「删了 1 万行」的 before/after |
| UT | `tests/dlt/ut/server/single_llm_req_handler/`（base/prefill/decode/pnd） |
| IT（多协议入口） | `tests/dlt/it/test_server_OpenAi.cpp`、`test_server_TGI.cpp`、`test_server_vLLM_OpenAi.cpp` 等 |

### 2.2 诚实话术（删行 / golden）

> Handler/Interface 分层和 UT/IT 目录我可以指到路径。简历「削减冗余约 1 万行」是重构合并 Prefill/Decode×多协议重复链路的**量级目标**；**重构前后 before/after cloc、PR 链接、合入日期本机证据包未钉死——标待补，上场不编数。**  
> 仓内**没有**独立叫 `golden_server_*` 的专用套件；回归讲 UT/IT 矩阵，不假装有黄金文件名。

| 待补物 | 用途 | 上场降级 |
|--------|------|----------|
| 重构 PR / change-id / 日期 | 外部可查 | 「开源/内网 PR 号在证据包；今晚未导出则只讲架构」 |
| before/after `git diff --stat` / cloc | 钉死「约 1 万行」 | 「数字以 PR diff 为准；现状 wc 只能证明分层还在」 |
| 合并前重复文件列表 | 讲清删的是复制粘贴 | 「删的是多协议×角色重复，不是业务蒸发」 |

## 3 · 5000+ 行拆解口述模板（模块占比「待 cloc」）

### 3.1 30 秒骨架（背）

> 「5000+ 钉在**结构化输出从 0 到 1**的累计交付，不是单文件刷行数。主干是 manager / grammar / bitmask / logits handler 接入，再加 plugin 异步时序与单测。Tool Call 和 Server 重构那『约 1 万行』**不混进**这个数字。精确 author 占比我用 `git log --stat` / cloc 补；你现在要听的是模块拆解与可指路径。」

| 桶 | 内容 | 现状锚点（可指） | 累计占比 |
