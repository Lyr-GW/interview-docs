## 1 · 60 秒电梯稿（可直接背）

> 量化三角：**显存 / 带宽 / 算力**。决策树收口：小 batch 延迟 → **W4A16**；大吞吐 → **FP8/W8A8**；极致压缩 → **FP4**；权重算法 **GPTQ vs AWQ**；KV 单独一支 `cache_dtype`+**FP32 acc**。[文档·01C/03]
>
> 精度不靠一句 PPL：三层验收——**任务指标**（bench/业务分）、**数值稳定**（NaN/Inf/logits 尖峰/层间 KL）、**长生成**（长依赖漂移、重复/崩溃）。掉点归因：关量化二分 → 分项开权/激/KV → 逐层找 outlier → 混精剔首尾/norm/router。[文档·03]
>
> 开关一句话：W8A8 吞吐开、延迟小 batch 慎；AWQ 权压开、激也要压另案；FP8 默认可试、acc 必须 FP32；KV FP8 为长上下文/slot 开，跨机 dtype/scale 不一致就关或先对齐契约。[文档·01C]

## 2 · 决策树收口（对照 01C）

```text
目标？
├─ 延迟敏感、小 batch → W4A16     # 压权省带宽；反量化税
├─ 吞吐 / 大 batch     → FP8/W8A8  # 低精 TC + 带宽双收
├─ 极致显存            → FP4/NVFP4 # 需 block scale；硬件叙事
└─ 权重量化选型        → GPTQ（最小重构误差） vs AWQ（护重要通道，LLM 更稳）
KV 旁支：cache_dtype=fp8；attention acc=FP32；CLI 记忆 --quantization/--kv-cache-dtype
```

| 口诀 | 勿反说 |
|------|--------|
| 小 batch → W4A16 | 大 batch 反量化税吃掉 TC 收益 |
| 大吞吐 → FP8 | 小 batch 未必吃满低精算力 |
| 拐点 | **未 profile 不报具体 batch 数** |

与 01C 关系：本卡 **不重背三角表**；上场先甩收口树，再进 §3 验收三层。

## 3 · 精度验收三层（本卡主线）

与 01C「烟雾→标准→业务」**互补**：那边是**流程闸门**，这边是**观测维度**——上场可先说维度，再映射闸门。

```text
L1 任务指标     → 能力有没有塌（分、通过率、业务 KPI）
L2 数值稳定     → 算得稳不稳（溢出、尖峰、层误差）
L3 长生成       → 跑得久会不会漂（长依赖、重复、中途崩）
```

| 层 | 查什么 | 典型信号 | 映射 01C 闸门 |
|----|--------|----------|---------------|
| 任务指标 | MMLU/GSM8K/HumanEval；业务 A/B | 数学/代码最敏感掉分 | L2 标准 + L3 业务 |
| 数值稳定 | NaN/Inf；logit max；层间 KL/余弦 | 偶发 Inf、某层 KL 爆 | 归因分项/逐层 |
| 长生成 | 长上下文续写、多轮 Agent、超长 decode | 前几千 token 正常、后段胡言/死循环 | 业务影子+金丝雀 |

**归因顺序（掉点时倒背）**

```text
全关量化复现基线 → 只开权 / 只开激 / 只开 KV
→ 逐层 KL或余弦定位 outlier
→ 混精：剔首尾、norm、MoE router 等敏感层
→ 上线：影子 → 金丝雀 → 可回滚
```

**金句**：PPL 烟雾过了 ≠ 三层都过；长生成挂了优先怀疑 **KV/累加**，任务分塌了优先怀疑 **权/激**——仍以分项 A/B 为准。[经验·01C]

## 4 · W8A8 / AWQ / FP8 / KV · 各一句开/关

| 项 | 何时开（一句） | 何时关/慎（一句） |
|----|----------------|-------------------|
| **W8A8** | 大 batch/吞吐、要低精 TC 与显存÷2 | 强延迟小 batch（更偏 W4A16）；无低精 TC 路径时 |
| **AWQ** | 权重量化、LLM 要稳、activation-aware | 只要「数学最小误差」对比 GPTQ 时改讲 GPTQ；激也要 4/8bit 另选型 |
| **FP8** | 新默认尝试：scale+粒度齐、训练衔接 | **acc 不能 FP32** 或 scale 未校准就别硬开 |
| **KV FP8** | 长上下文要 slot、decode 带宽紧 | prefix/PD **dtype·scale 契约不齐**→先关或对齐，防假命中/错反量化 |

FP8 四要素速记（开之前默念）：**scale / 粒度 / KV dtype / acc=FP32**。[文档·03]

## 5 · 快问 8 题（10–20s / 题）

1. **决策树三叉？** → 小 batch W4A16；吞吐 FP8/W8A8；极致 FP4。  
2. **验收三层维度？** → 任务指标 / 数值稳定 / 长生成。  
3. **与烟雾→bench→业务关系？** → 维度 vs 闸门；互补不互替。  
4. **W8A8 何时慎？** → 延迟敏感小 batch，反量化/算力路径不划算。  
5. **AWQ vs GPTQ？** → AWQ 护通道更稳；GPTQ 最小重构误差。  
6. **FP8 为啥要 FP32 acc？** → 低精累加误差爆；输出再 clamp。  
7. **KV 开了跨机注意？** → dtype/scale 一致；否则假命中或错解。  
8. **掉点怎么拆？** → 二分→分项权/激/KV→逐层→混精排除。

## 6 · 追问 3 连（严格面试官）

**连 1 ·「任务指标过了，还要做数值稳定和长生成吗？」**  
→ 要。短 prompt bench 可掩盖累加漂移与 KV 误差；Agent/长文档是另一失败面。三层是**并联闸**，不是「分够了就上」。[机制·经验]

**连 2 ·「长生成崩了，你为什么先动 KV 而不是 AWQ？」**  
→ 经验先验：权/激更伤短题 logits 尖峰；KV 更伤长依赖与 cache 带宽路径——**先验只用于排优先级，结论必须分项 A/B**。没有分项表就承认「先复现再定责」。[经验·01C]

**连 3 ·「PD + KV FP8，验收多哪一条？」**  
→ 在三层之上加 **契约验收**：P/D 与亲和路径上量化字节+scale 一致；错契约会表现成「指标偶发傻」或亲和假命中。未读 connector 字段时不编包格式，只讲一致性。[机制·01C/12]
