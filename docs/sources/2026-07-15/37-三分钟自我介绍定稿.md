## 1 · 90 秒版（默认开场 · 可直接背）

> 我目前在华为昇腾做 MindIE LLM **推理服务**，主线是把特性从 0 推到可维护交付。三条线：  
> **一、结构化输出**：Schema → xgrammar 字节级 PDA → bitmask → 采样硬约束；编译缓存是 **FIFO、默认 100**。异步调度下 mask 错位我修过，验收钉在正确性。bitmask 落地是 **torch NPU 算子组合**，不是自研 fused kernel。  
> **二、Motor KV 亲和**：Coordinator **同源 tokenize** → Conductor 精确最长前缀 → unified/load_gated；相对字符级近似树，我们是 **查真索引 + 驱逐事件**。主收益是少 Prefill；简历 TTFT−70% 是 **代表性测算**，不是甩客户 raw log。  
> **三、Tool Call**：协议适配 + 流式解析（Encode/Generate/Decode）；和结构化硬约束是不同保证层级，集成边界我讲得清。旁路有 Server Handler/Interface 重构——路径可指，LOC 待补不编。  
> 今天希望深聊推理加速与调度；你可以从简历任意一条往下挖。

**时长自检**：约 85–95s；三条各 ~25s；结尾交棒一句。

## 2 · 180 秒版（「多讲讲 / 三分钟」· 可直接背）

> 我目前在华为昇腾做 MindIE LLM **推理服务**开发，入职约一年，主线是服务侧系统问题：延迟、吞吐、缓存命中、正确性。三条可核交付：  
>  
> **结构化输出 0→1**：从 Schema 编译到采样硬约束整条链路我扛过——xgrammar 做字节级 PDA，再落到 bitmask 约束 logits。编译缓存按内容哈希，策略是 **FIFO、默认 100**，命中不调序。异步调度场景里出现过 mask 与 token 游标错位导致非法 JSON，我按线程/游标/顺序三因修过，验收是正确性，不拿未测 TPOT 百分比圆场。bitmask 在 NPU 上是 **torch 组合 `masked_fill_`**；fused 是可优化点，**我不上场吹自研 kernel**。线上落地后端是 xgrammar；guidance 多后端是预留抽象，**不是「线上有 guidance」**。MTP 与结构化在入口 **硬互斥**，不能一起开。个人累计交付量级简历写 5000+，钉在这条线，不和 Tool/Server 行数混算。  
>  
> **Motor KV 亲和**：问题是多实例下「猜缓存 vs 查缓存」。我们做 token 级精确亲和——先同源 tokenize（含 tools 透传、fail-closed），再查 Mooncake Conductor 全局索引，打分用 unified/load_gated，并演进过防 herding。相对 SGLang/vLLM Router 本地字符树，我们错向偏向安全假阴。PD 下 **D 实例不注册** Conductor，只 P/U 上报。TTFT−70% 是按命中少 Prefill 的 **代表性测算**，公式与假设可讲；**不是客户原始日志实测**。Mooncake 我用边界讲：Conductor 查、Store 放、TE 传；通信/HCCL 细节 **非主战场**，分层正交即可。  
>  
> **Tool Call + Server**：Tool Call 侧重协议适配与流式状态机解析，和结构化「生成期硬约束」互补；MindIE 上 structural tag 一类深度集成边界如实讲。Server 侧把 Handler（引擎/PD）与 Interface（协议/流式）拆开，合并的是复制粘贴链路；「约 1 万行」以 PR diff 为准，**今晚不编 before/after**。  
>  
> 相对最熟是 **KV 亲和 + 结构化 0→1**；算子/投机我用同套归因语言协作，系统跟踪还在补。希望今天往调度、约束解码或 PD 任一方向深挖。

**时长自检**：约 2:40–3:00；SO ~70s / 亲和 ~55s / Tool+收尾 ~35s。

## 3 · 结构骨架（忘词时按格填）

```text
身份：昇腾 · MindIE · 推理服务 · 0→1 交付
  ├─ SO：PDA→bitmask→采样 · FIFO/100 · async 错位 · torch 非 fused
  ├─ 亲和：tokenize→Conductor→门控 · 查≠猜 · 测算70% · D不注册
  └─ Tool：协议+流式解析 · （可选）Server 边界 · LOC待补
交棒：可从简历任一条挖
```

**90s 砍法**：每条只留「做了什么 + 一个红线钉子」；砍掉 MTP/guidance/herding/Server 展开。  
**180s 加法**：每条补「失败模式或边界一句」+ 明确「最熟两块」。
