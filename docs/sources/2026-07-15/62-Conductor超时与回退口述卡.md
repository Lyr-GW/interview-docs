## 1 · 60 秒电梯稿（可直接背）

> Motor 选中心化 Conductor 查真索引，就要付「一跳 HTTP + 组件可用性」——设计原则是 **亲和是优化不是依赖**（N2）。查询客户端超时 **0.2s（200ms）**：**[配置]**，不是已测 P99。超时/无数据/tokenize fail-closed → 整条亲和返回 None，调度 **回退 LoadBalance**，请求主路径不堵。[文档·ir/12]
>
> LoadBalance **不是轮询**：全局最小负载（`heapq.nsmallest`），负载分与 unified 第二项同源；回退=去掉亲和项、负载视角连续。再失败可落到 round_robin。D/E 非 KVA 角色本来就走 LB。[文档·ir/12 Q4]
>
> 常态目标：Conductor **P50&lt;5ms / P99&lt;20ms**——**[目标·非实测]**；0.2s 是硬上界非常态。无数只报「配置超时 + 回退 LB」，不编分位。[15 E3 / 50]

**金句**：超时配 200ms；超则 LB；亲和失败不拖垮路由。

## 3 · 失败怎么回退（不拖垮路由）

```text
触发（任一）:
  Conductor 超时 / 挂 / 无匹配数据
  tokenize fail-closed → []
       ↓
亲和路径返回 None
       ↓
LoadBalance（最小负载，非 RR）
       ↓（再异常）
round_robin 兜底
```

| 层 | 行为 | 标注 |
|----|------|------|
| 亲和 | 快速失败，**不阻塞**生成主路径 | [机制·N2] |
| LB | 去掉 matched/prefill_cost，只比负载 | [代码事实] |
| 平局 | `start_index` 按 worker 旋转，冷启动少齐刷 | [机制] |
| 仲裁 | LB 候选仍走 worker 提案 + Scheduler 权威 | [机制·与亲和同基建] |
| Conductor 重启 | `/services` 对账重注册 + kv-events replay | [机制] |

**与假命中一句**：残缺窗口错向偏**假阴**→安全退 LB；不是硬猜仍命中。[12]

## 4 · LoadBalance 三句（被追「回退到什么」）

1. **不是 RR**：摊平所有 endpoint，取负载分最低者。  
2. **复合分**：`endpoint_score + weight × (instance_score / endpoint_count)`；`endpoint_instance_score_weight` 默认 **0.05** **[配置]**——感知「同机其他 DP 已满」。  
3. **与 unified 连续**：LB = unified 去掉亲和项；量纲仍是 token 负载分。

## 6 · 追问 3 连

**① 「200ms 是不是太大了？会不会拖 TTFT？」**  
→ 健康路径应远小于 200ms（目标 P99≪上界）。200ms 是**故障快失败**预算：宁可偶发放弃亲和，也不让慢 Conductor 把调度打成秒级。真拖 TTFT 的是超时后走满量 prefill，不是超时值本身——所以要盯回退率与命中率，而不是把超时调成「假装很低的 P99」。[配置 vs 目标]

**② 「回退 LB 会不会制造 herding？」**  
→ LB 仍有最小负载 + 平局旋转 + Scheduler 权威；比「假阳性齐灌空壳机」安全。亲和 herding 另用 overlay→top-k→unified 演进修（见 `17`）；超时回退是可用性层，不是打分层。[ir/12 + 17]

**③ 「为何不本地树省掉这一跳？」**  
→ 本地字符树省 RTT，但无驱逐感知、易假阳灌热点。中心多付毫秒级查询，换精确命中；超时把可用性风险封顶。选型判断：错一个 block = 成百上千 token 重算，**精度比省一跳值钱**。[12 / ir/12 §1.1]
