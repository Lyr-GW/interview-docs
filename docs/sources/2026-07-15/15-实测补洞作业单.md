## E1 · 结构化输出 TPOT A/B（P0）

**目的**：补二面「开约束后速度？」；把「估计 <1%–3%」升级为可出示区间，或诚实保留估计。

**最小实验设计**：固定模型/并行/并发/`max_tokens`/同一 Schema → warmup ≥50（FIFO/100 热）→ **A** 无 `response_format` vs **B** 开 xgrammar → 每臂 ≥200 完成请求 → 主指标 TPOT P50/P99，辅 TTFT 冷/热、TPS。只改约束开关。

| B xgrammar | | | | | | | |
| Δ (B−A)/A | — | % | % | | | | 只填实测 |

**过关**：☐ 同负载+warmup　☐ 能口述 ΔTPOT P50（或样本不足不报）　☐ 冷/热 TTFT 分开　☐ 估计≠客户实测  

**无环境降级**：交付时无严格 A/B；热路径是 bitmask + `masked_fill_`，相对 FA 应是小头。只报估计量级或补测区间，不编精确百分点。

## E2 · 投机接受率–并发曲线（P0）

**目的**：用曲线支撑「大 batch / GPU 不闲时投机失效」，避免只背定性句。

**最小实验设计**：固定 draft 配置与 prompt 集 → 至少 **2 点**（低并发有空泡 + 高并发/大 batch 近 compute-bound）→ 记 τ 或 accept rate、TPS/TPOT、相对关投机加速比 → 观察高负载 τ↓ 或加速比→1。

| 点 | 并发/batch | 接受率或 τ | TPS | vs 关投机 | GPU 观感 | 备注 |
|----|------------|------------|-----|-----------|----------|------|
| L 低负载 | | | | | 空/中 | |
| H 高负载 | | | | | 满 | |
| （可选）M | | | | | | |

**过关**：☐ ≥2 点能讲清失效　☐ 区分「接受率低」vs「算力不闲」　☐ 无曲线不报自制加速比  

**无环境降级**：机制上接受率不够高或 GPU 不闲就会失效甚至负收益；曲线未跑完则不上场报加速比。
