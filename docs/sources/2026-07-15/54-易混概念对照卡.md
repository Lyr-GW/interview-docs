## 1 · 八组对照（上场抄板）

### 1 · PDA vs FSM

| | |
|--|--|
| **一句话** | xgrammar 约束是 **字节级 PDA**（可递归嵌套括号/结构）；随口说 **FSM** 过窄——正则/扁平状态机盖不住 JSON Schema 嵌套。 |
| **挖穿** | 「PDA 和 FSM 差在哪？」「你们仓里写过 FSM 吗？」→ 答「差不多」或坚持 FSM → 对不上专题/`03` #4。 |
| **翻卡** | 本夜 `03` #4；深文 `ir/03`、`ir/16` |

### 2 · FIFO vs LRU（编译缓存）

| | |
|--|--|
| **一句话** | MindIE schema 编译缓存默认 **100 条、FIFO、命中不调序**；**不是** LRU/128（旧口述地雷）。 |
| **挖穿** | 「命中会不会挪到队尾？」「容量多少？」→ 答 LRU/「热 schema 更抗踢」→ 与 `structured_output_manager` 源码冲突。 |
| **翻卡** | `03` #1；白板图 `52` §5；抽检 `40` |

### 3 · 字符级 vs token 级匹配

| | |
|--|--|
| **一句话** | vLLM Router 本地树按 **字符**猜最长前缀；Motor **tokenize 后**按 **token ID** 查 Conductor——tools/chat-template 下二者可错位。 |
| **挖穿** | 「字符前缀长就一定 token 可复用？」→ 答「是」→ 被 tools 注入打穿；再问「上层 tokenizer 哪来」→ 须答 model_path 同源。 |
| **翻卡** | `12` · `18`；对标一句 `26` |

### 4 · handoff vs concurrent（PD）

| | |
|--|--|
| **一句话** | **handoff**：P 做完+KV 就绪再调 D，TTFT≈\(T_p+T_{tx}\)；**concurrent/layerwise**：P/D 同时起、逐层 sync，TTFT≈\(\max(T_p,T_{tx})\)——前提是 connector **真支持**逐层。 |
| **挖穿** | 「你们一定是流式逐层吧？」→ 空吹 layerwise；或把 PD 主收益说成「传一定比算快」→ 临界点白板穿。 |
| **翻卡** | `07`；选型叠 `24` |

### 5 · approximate vs precise 亲和

| | |
|--|--|
| **一句话** | **Approximate**=路由器影子树/历史猜谁有；**Precise**=引擎事件养真值索引再 `/query`——猜可假阳灌热，查错向偏安全假阴。 |
| **挖穿** | 「你们和 SGLang cache_aware 一样吗？」→ 答「都是最长前缀」不拆档 → 假阳 vs RR 故事讲不出。 |
| **翻卡** | `12` · `26`；深文 `ir/15` |

### 6 · APC vs 跨实例亲和

| | |
|--|--|
| **一句话** | **APC**=同实例内满块 hash 链自动跳过 prefill；**跨实例亲和**=路由把请求送到「存货最长」的机——**两层正交**，落点后仍靠 APC 真省算。 |
| **挖穿** | 「开了 prefix cache 还要 Motor 干嘛？」→ 答「一样」→ 多 replica 打散稀释讲不清；或把 ZMQ 说成搬 KV。 |
| **翻卡** | `27`；事件面 `25`；索引层 `13` |

### 7 · Conductor vs Store vs TE

| | |
|--|--|
| **一句话** | **Conductor** 查谁有前缀+选路（不搬字节）；**Store** 管放哪/淘汰；**TE** 管怎么搬（RDMA/TCP…）。Motor 只对接 Conductor 决策。 |
| **挖穿** | 「Mooncake 底层怎么实现？」→ 揉成「一个存 KV 的东西」或把 Connector 说成 Mooncake 官方组件 → 二面 Q7 复现。 |
| **翻卡** | `13`；传边界 `36`；深文 `ir/04`/`10` |

### 8 · 约束解码 vs tool parser

| | |
|--|--|
| **一句话** | **约束**限制采样「下一 token 合法」；**parser** 在 Decode 抽 name/arguments、管流式 delta——**正交、都要跑**；开 xgrammar ≠ 可省 ToolCallsProcessor。 |
| **挖穿** | 「你们约束了还要 parser？」→ 答「不用」；或把 Hard Cut-off 说成全模型族默认 stop → `28` 穿。 |
| **翻卡** | `28`；综合 `ir/17`；MTP 叠约束 → `23` |

## 3 · 30 秒自检（睡前）

1. 编译缓存？→ **FIFO / 100**，非 LRU  
2. xgrammar？→ **字节级 PDA**，非随口 FSM  
3. Motor vs Router？→ **token 查** vs **字符猜**  
4. PD 两种切换？→ handoff 加和 / concurrent 取 max  
5. APC 够不够？→ **不够**，跨机还要亲和  
6. Mooncake 三层？→ 查 / 放 / 搬  
7. Tool 开约束？→ **parser 仍要**  
8. D 实例？→ **不注册** Conductor（旁挂 `03` #6，易与「谁上报索引」混）
