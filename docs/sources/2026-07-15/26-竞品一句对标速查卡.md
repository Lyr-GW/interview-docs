## 1 · 60 秒「生态地图」（可直接背）

> 亲和四档记忆：Sticky 跟会话 → Hash 跟桶 → **Approximate 跟路由器影子树** → **Precise 跟引擎真值事件**。[文档·15]
>
> 业界默认开口往往是 **SGLang `cache_aware` / vLLM Router**：本地 **字符级** 近似树，零同步、不知驱逐——**猜缓存**。我们 Motor 是 **tokenize → Mooncake Conductor `/query` → unified/load_gated**——**查缓存**；索引来自 ZMQ `BlockStored/Removed`，错向偏向安全假阴。[文档·12 / 15]
>
> **Dynamo**：全栈 runtime，cost 公式里写死 tier 权重，默认 precise 事件，可退 approximate+TTL——**同一问题、精确解法不同形态**。[文档·01A]  
> **llm-d**：K8s EPP 插件双轨（approx LRU / precise Indexer）；Mooncake Store **可选**生产参考，PD 传数走 **NIXL**。[文档·19]  
> **AIBrix**：自研 L1/L2 + 默认近似；**几乎不用 Mooncake**——反证「池化≠Mooncake」。[文档·19]  
> Mooncake 自身是 **可插拔三层**（TE/Store/Conductor），不是行业唯一默认。[文档·13]

**金句（倒背）**：「他们猜缓存；我们查缓存。」

## 2 · 一行对标表（上场抄板）

| 名字 | 定位一句话 | 精确/近似 | 与 Mooncake | 何时提 |
|------|------------|-----------|-------------|--------|
| **Motor** | tokenize→Conductor `/query`→负载融合选机 | **精确**（D） | 控面强依赖 Conductor；存/传在 Connector | 简历主线；「差在哪」锚 |
| **SGLang cache_aware** | 网关本地字符 radix，按历史猜最长前缀 | **近似**（C）；实验才有 ZMQ | L3 backend **之一**；Gateway 不知 L2/L3 | 「业界默认」；假阳比 RR 糟 |
| **vLLM Router** | SGLang Gateway 的 Rust fork；算法同源 | **近似**；官方无内置精确索引 | P/D 编排可接 Mooncake≠精确亲和 | 命名拆坑；「官方是不是精确」 |
| **Dynamo** | Frontend+KV Router+KVBM G1–G4+NIXL | 默认 **精确**；可 `--no-router-kv-events`→近似+TTL | KVBM **不依赖** MC；SGLang 旁支可选 | 「和 Dynamo 比」；cost 白板 |
| **llm-d** | EPP：Filter→Score→Pick 多插件加权 | **双轨** C/D 用户自选 | Store 生产可选；**PD=NIXL 不用 MooncakeConnector** | K8s 标准化；「是不是都用 MC」 |
| **AIBrix** | Gateway prefix-cache + 自研 L1/L2 CRD | 默认 **近似**；Event Sync 可升、Cleared 空 | **几乎不用**；同名 conductor≠MC | 「池化=Mooncake？」反例 |
| **Mooncake** | TE 搬 / Store 放 / Conductor 查+选 | Conductor 侧 **精确** | 自身；市场=可选 backend | 「一分钟分层」；Motor 边界 |

## 3 · 每行扩写（30s / 家，点名才展开）

### 3.1 Motor（原点）

> Precise lookup：无本地 prefix 树；D **不注册**；超时 0.2s→回退 LB；缺 utilization/speculative 诚实承认。[文档·12 / 03]

### 3.2 SGLang `cache_aware`

> 树键=`raw text`，刻意不 tokenize；自管 LRU，**不知引擎驱逐**。失衡→最短队列；否则 match_rate 过阈打命中机。字符≠token（tools/template 易级联 miss）。[文档·15 / 01]

### 3.3 vLLM Router

> **命名地图**：`sgl-model-gateway` → fork → `vllm-project/router`（Rust）；**≠** production-stack 里 Python `vllm_router`。换 fork **不会**自动变 D 层；精确在 llm-d / Dynamo / `kvaware`(LMCache) 等别处。[文档·15]

### 3.4 Dynamo（接 `01`§A）

```text
cost = scale × max(raw_prefill − overlap_credit, 0) + decode_blocks
credit ≈ 1.0·device + 0.75·host + 0.25·disk (+ shared 若开)
→ min(cost)
```

> 统一栈把亲和与分层惩罚写进同一公式；我们是 Conductor 外挂精确查询——形态不同，问题同一。[文档·01A]

### 3.5 llm-d

> `prefix-cache-scorer` 权重大 + util/queue；precise 路径要 **同源 tokenize + PYTHONHASHSEED 一致**，否则静默 0 命中。Store≠PD Connector。[文档·19 / 08]

### 3.6 AIBrix

> 路由与 offload **解耦**；L2 列表无 Mooncake；`conductorPrefillPolicy` **命名撞车、代码独立**。负载门控偏 running imbalance，非 utilization。[文档·19]

### 3.7 Mooncake（中立基建）

> Conductor 不搬 KV；Store 不做全局路由；Connector **不是** MC 官方组件。Motor 只决策。[文档·13]

## 5 · 快问 8 题（10–20s / 题）

1. **猜 vs 查？** → 近似树猜谁有；Conductor 查引擎真值。  
2. **vLLM Router 精确吗？** → **否**（默认字符近似）；fork≠精确。  
3. **Dynamo 依赖 Mooncake？** → **否**；KVBM 自研；MC 仅旁支可选。  
4. **llm-d PD 用 MooncakeConnector？** → **否**；统一 NIXL；Store 另说。  
5. **AIBrix conductor？** → 同名思路，**不是** Mooncake Conductor。  
6. **假阳谁更糟？** → 无 Removed 的近似树可比 RR 糟（见 `12`）。  
7. **Motor 有本地树吗？** → **无**；只 `/query`。  
8. **Mooncake 是行业默认？** → **否**；可插拔 backend 之一。

## 6 · 红线 8 条（页脚倒背）

1. approximate ≠ precise；换 Router fork 不变 D。  
2. 字符树假阳无界；Conductor 残缺主风险偏假阴。  
3. 假阳性可比 RR 更糟；测算数一律标测算。  
4. Mooncake ≠ 唯一；AIBrix 几乎不用；Dynamo KVBM ≠ MC。  
5. StoreConnector ≠ Connector；llm-d PD = NIXL。  
6. AIBrix conductor ≠ Mooncake Conductor。  
7. NIXL ≠ 前缀索引。  
8. Motor：D 不注册；无本地树；不报未测加速比。
