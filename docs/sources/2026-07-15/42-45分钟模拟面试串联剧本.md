## 1 · 自我介绍（0–3′）

| 项 | 内容 |
|----|------|
| **时限** | 3′（考生默认 90s；对方说「多讲讲」才放 180s） |
| **主问** | 「先介绍一下你自己，以及你最近在做的推理相关工作。」 |
| **预期翻卡** | [`37`](./37-三分钟自我介绍定稿.md)；骨架急救 [`20`](./20-明早上场作战手册.md) §1.2 |
| **通过标准** | ① 三条线可辨：SO / KV 亲和 / Tool（或 Server 半句）② 开口嵌 **FIFO/100** 与 **测算**（或等价红线）③ 不吹自研 fused / 客户 raw log ④ 结尾交棒「可从简历往下挖」 |
| **故意高压追问 ×1** | 「你简历写 TTFT 降 70%，是客户线上实测吧？」→ 必须立刻改口「代表性测算」；模板见 [`39`](./39-高压追问应急话术卡.md) 场景「要客户日志」 |

**面试官听点**：钩子是否清晰（下一刀砍哪条）；数字是否可辩护。  
**考生急救**：忘词只背 `37` 90s；禁 LRU/128、禁「实测 −70%」。

## 2 · 结构化输出深挖（3–13′）

| 项 | 内容 |
|----|------|
| **时限** | 10′（前 4′ 链路；中 4′ 工程坑；后 2′ 互斥/边界） |
| **主问** | 「结构化输出你们怎么做的？从 Schema 到采样讲清楚；编译缓存策略是什么？」 |
| **预期翻卡** | 链路/`02`§1；缓存红线 [`03`](./03-口径红线速查卡.md)；async [`16`](./16-异步调度mask错位口述卡.md)；bitmask [`19`](./19-bitmask-NPU路径诚实卡.md)；MTP×SO [`23`](./23-MTP与结构化互斥深挖卡.md)；总览旁链 [`01`](./01-P0口述卡-Dynamo投机量化Profiling.md) |
| **通过标准** | ① Schema→EBNF→**字节级 PDA**→bitmask→采样 ② 缓存 **FIFO / 默认 100**，命中不调序 ③ bitmask=**torch 组合**，非自研 kernel ④ 被问 MTP：入口**硬互斥** ⑤ guidance=预留，线上仅 xgrammar |
| **故意高压追问 ×1** | 「你们文档/旧口述写过 LRU 128，你现在说 FIFO 100，到底哪个真？」→ 立刻钉代码真相 FIFO/100；承认旧口径过时；见 `39`「简历数字对不上」 |

**面试官可加刀（可选，超时则砍）**：async mask 错位三因；5000+ 别混 Tool/Server 行数。  
**考生急救**：被抓 LRU → 不辩解「差不多」，直接订正 + 源码层策略一句。

## 3 · KV 亲和（13–25′）

| 项 | 内容 |
|----|------|
| **时限** | 12′（机制 5′ → 假命中/事件 4′ → 测算/Herding 3′） |
| **主问** | 「Motor KV 亲和相对 SGLang/vLLM Router 本地树，核心差在哪？假命中怎么防？」 |
| **预期翻卡** | 假命中 [`12`](./12-假命中与驱逐感知口述卡.md)；Herding [`17`](./17-Herding与负载门控口述卡.md)；Tokenizer [`18`](./18-Tokenizer同源与tools透传口述卡.md)；ZMQ [`25`](./25-ZMQ-KV-Events速答卡.md)；TTFT [`38`](./38-TTFT五段分解与70%测算卡.md)；Mooncake 三层 [`13`](./13-Mooncake三层60秒口述卡.md) |
| **通过标准** | ① tokenize 前置 + Conductor **精确最长前缀**（查索引 vs 猜缓存）② 假阳 vs 假阴取舍；ZMQ **Removed** 驱逐感知 ③ **D 不注册** Conductor ④ −70% 能白板假设或标「测算」⑤ Herding：overlay→top-k→unified 演进一句 |
| **故意高压追问 ×1** | 「Conductor 挂了 / 超时你怎么办？你们 P99 多少？」→ 讲 0.2s 超时+回退；**无 P99 不报数**（待补见 `15`）；高压模板 `39`「没测过」 |

**面试官听点**：通知面（ZMQ）≠ 数据面（TE/Store）；勿把 Connector 说成 Mooncake 官方三层。  
**考生急救**：被逼客户日志 → 翻 `38` 测算假设清单，不甩 raw log。

## 4 · 投机解码白板（25–33′）

| 项 | 内容 |
|----|------|
| **时限** | 8′（白板 5′ + 演进/失效 3′） |
| **主问** | 「白板画一下投机解码的拒绝采样；什么叫无损？大 batch 为什么容易掉？」 |
| **预期翻卡** | 白板核 [`32`](./32-拒绝采样白板特训卡.md)；口述母本 [`01`](./01-P0口述卡-Dynamo投机量化Profiling.md) B；演进 [`34`](./34-投机演进线默背卡.md)；深文 `ir/02` |
| **通过标准** | ① draft→verify→accept/reject **四步**可画 ② 「无损」=与目标分布一致（非「一定更快」）③ 大 batch / 低接受率失效一句 ④ 与结构化：**硬互斥**（若追问）⑤ 无压测曲线**不报加速比** |
| **故意高压追问 ×1** | 「DSpark / 上周那篇你们线上开了吗？加速比多少？」→ 诚实：跟论文/演进线，**未生产压测不报百分点**；迁移 draft-verify + 接受率；见 `39`「追加速比」+ `01` B |

**面试官听点**：会不会只背名词不会画；会不会用假加速比填沉默。  
**考生急救**：白板卡壳 → 先写四步框再填细节；演进用 `34` 每代一句即可。

| **主问** | 「如果目标是降 TTFT / 提吞吐 / 保正确性，你分别会动哪些 vLLM（或同构）配置？为什么？」 |
| **预期翻卡** | [`09`](./09-vLLM配置背后原理串讲卡.md)；Scheduler/CB [`11`](./11-Scheduler与ContinuousBatching口述卡.md)；深文 `ir/05` |
