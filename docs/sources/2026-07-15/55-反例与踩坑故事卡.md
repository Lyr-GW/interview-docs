## 1 · 十例反例库

### ① 跳槽动机：涨薪 + 嫌加班

| | |
|--|--|
| **错答** | 「行情好想涨点薪；华为加班、周末加班比较多。」（二面 Q2 原文形态） |
| **为什么砸** | 拉力=0、推力拉满；后面「几个月就跳」会被钉死在钱/苦，难翻技术主线。 |
| **正答锚点** | 拉力>推力：推理栈深度 / 端到端特性交付空间；华为成长一句肯定。全文 [`08`](./08-软性题跳槽动机对练.md)；高压 [`39`](./39-高压追问应急话术卡.md) §跳槽 |

### ② Mooncake：「只会用、底层不懂」

| | |
|--|--|
| **错答** | 「底层我可能不是特别了解。」（Q7）或揉成「就是个存 KV 状态的东西」。 |
| **为什么砸** | 简历写了 Conductor/亲和，底层零分层 → 证明「配置工」嫌疑；Q6 浅、Q7 空是连续失分。 |
| **正答锚点** | 三层：Conductor 查+选 / Store 放 / TE 搬；Motor 只决策。翻 [`13`](./13-Mooncake三层60秒口述卡.md)；传边界 [`36`](./36-HCCL与KV传输边界卡.md) |

### ③ vLLM 配置：「一下子想不到」

| | |
|--|--|
| **错答** | 「一下子想不到。」（Q17 零输出） |
| **为什么砸** | 日常搭 Motor+vLLM 却说不出一类加速旋钮 → 面试官判定「不会迁移到配置层」。 |
| **正答锚点** | 按目标选：prefix / chunked / CG·aclgraph / 投机 / KV dtype / 预算… 串讲 [`09`](./09-vLLM配置背后原理串讲卡.md)；题库 `ir/05` |

### ④ 投机：「EAGLE 忘了 / DSpark 没看」

| | |
|--|--|
| **错答** | 「EAGLE 有点忘记了。」「新论文还没了解原理。」（Q12/Q15） |
| **为什么砸** | 自称看过投机却断在演进线与拒绝采样白板 → 最大知识失分区。 |
| **正答锚点** | 演进默背 [`34`](./34-投机演进线默背卡.md)；拒绝采样四步 [`32`](./32-拒绝采样白板特训卡.md)；母本 `01` B / `ir/02` |

### ⑤ 编译缓存：LRU / 128

| | |
|--|--|
| **错答** | 「编译缓存是 LRU，大概 128。」 / 「命中会挪到最近。」 |
| **为什么砸** | 与源码 FIFO/100、命中不调序冲突；本夜最高危口径，翻仓即穿。 |
| **正答锚点** | **FIFO / 默认 100 / 命中不调序**。红线 [`03`](./03-口径红线速查卡.md) #1；易混 [`54`](./54-易混概念对照卡.md) §2；抽检 [`40`](./40-口径全仓抽检报告.md) |

### ⑥ TTFT−70%：「客户日志实测」

| | |
|--|--|
| **错答** | 「客户场景实测 TTFT 降了 70%。」+ 拒给日志仍死撑实测。 |
| **为什么砸** | 要 raw log / 假设清单时无法出示 → 数字信用崩盘，连带亲和故事打折。 |
| **正答锚点** | **代表性测算**；五段 + 假设表。[`38`](./38-TTFT五段分解与70%测算卡.md)；诚实 [`02`](./02-简历第三层追问弹药.md)/[`50`](./50-简历数字诚实总表.md)；`03` #7 |

### ⑦ 约束自动机：随口 FSM

| | |
|--|--|
| **错答** | 「就是个 FSM 约束 next token。」 |
| **为什么砸** | Schema 嵌套要栈；红线写清 **字节级 PDA**；FSM 会被追「正则够不够」。 |
| **正答锚点** | **字节级 PDA**（xgrammar）。`03` #4；深文 `ir/03`；对照 [`54`](./54-易混概念对照卡.md) §1 |

### ⑧ D 实例也注册 Conductor

| | |
|--|--|
| **错答** | 「P/D 都上报索引，D 也能被亲和选中。」 |
| **为什么砸** | 与 `_KVA_ROLES={P,U}` 不符；污染「谁写了可复用前缀」语义。 |
| **正答锚点** | **D 不注册**；亲和只选 P/U。`03` #6；[`07`](./07-PD分离handoff口述卡.md) §4；[`24`](./24-PD混部与分离选型口述卡.md) |

### ⑨「开了约束就不用 tool parser」

| | |
|--|--|
| **错答** | 「上了 xgrammar / structural tag，Decode 直接合法，parser 可省。」 |
| **为什么砸** | 约束≠字段组装与流式 4-Case；Hard Cut-off、finish_reason 仍在 Decode。 |
| **正答锚点** | 约束⊥parser，**都要跑**。[`28`](./28-FunctionCall快问加固卡.md)；`ir/17` |

### ⑩「APC / prefix cache = 跨实例亲和」

| | |
|--|--|
| **错答** | 「开了 automatic prefix cache 就等于多机 KV 亲和。」 |
| **为什么砸** | APC 只救同实例；多 replica 乱打仍稀释；ZMQ 是通知不是搬块。 |
| **正答锚点** | 两层正交：APC 本机跳算 + Motor/Conductor 选机。[`27`](./27-引擎PrefixCache内核口述卡.md)；假阳 [`12`](./12-假命中与驱逐感知口述卡.md) |
