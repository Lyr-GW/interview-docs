## 1 · 60 秒电梯稿（可直接背）

> KV 亲和打分五旋钮：`mode` 选软融合还是硬门控；其余四系数只在对应语义里生效。默认 **`unified`**，三权重全 **1.0**，`load_gate_topn` 字面 **0→语义 2**。[配置默认·17]
>
> **unified**（默写）：  
> `score = prefill_load_scale × max(0, isl − overlap_credit × matched) + load_weight × load`，取 **min**。量纲全是 token；命中够大可翻盘负载。[机制·ir/12]
>
> **load_gated**（默写）：① 负载升序取 Top-N ② N 内 `matched` 降序（平局更低负载）。亲和**出不了**最闲集合——有限 `load_weight` 表达不了这硬上界，故必须单开模式。[17]

**金句**：软和可翻盘；硬门控不出 Top-N；默认 unified / 1.0 / topn→2。

## 3 · 公式骨架（可默写白板）

### 3.1 unified（默认 · 软融合）

```text
prefill_cost = max(0, isl − overlap_credit × matched)
score        = prefill_load_scale × prefill_cost + load_weight × load
选 endpoint:  argmin(score)   // 平局偏低 prefill_cost（更好亲和）
```

| 性质 | 一句 |
|------|------|
| 量纲 | 两项都是 **token** |
| 翻盘 | **能**：命中大 → prefill_cost 小 → 可压过高 load |
| Herding | unified 升 **V3** 全量全局重排（#304） |
| 退化 | `load_weight=0` → 纯最长前缀 |

### 3.2 load_gated（硬门控）

```text
① candidates = nsmallest(N=load_gate_topn, endpoints by load)
② 在 candidates 内: matched 降序；平局取更低 load
禁止: 亲和把请求拉出「最闲 Top-N」
```

| 性质 | 一句 |
|------|------|
| 亲和 vs 负载 | 亲和**永远不能**压过门控 |
| Herding | **钉 V2** top-k；故意不带 `prefill_cost` 软重排 |
| 退化 | `topn=1` → 近似纯 LB |
| 为何单开模式 | 字典序硬约束 ≠ 任何有限加权和 |

口述禁语：

> 「把 `load_weight` 调很大就能冒充 load_gated」——错；weight→∞ 会压死亲和变纯 LB，仍不是「先负载再亲和」的字典序。

## 4 · 翻盘临界（测算例 · 可算账）

锚：isl=8192、matched=6000、三系数全 1.0。[测算·17/13]

```text
epA（高命中）: score = (8192−6000) + load_A = 2192 + load_A
epB（零命中）: score = 8192 + load_B
epB 胜 ⟺ load_A − load_B > 6000   (= overlap_credit × matched)
```

调参直觉：↑credit → 空闲更难翻盘；↑load_weight → 空闲更容易赢；↑scale → 命中优势更响。均为**机制账**，非压测曲线。

选型：`unified` = 追命中可翻盘；`load_gated` = 控长尾护 Decode。打分再精，假阳仍靠 ZMQ Removed（`12`）；utilization/speculative **未进分**。

## 6 · 追问 3 连

**① 「和 Dynamo overlap 一回事吗？」**  
→ 直觉同类；Motor 单层 `credit×matched`，Dynamo 还有 Device/Host/Disk/Shared 分层。先默 Motor，点名再展开（`01` A）。

**② 「为何 gated 不升 V3？」**  
→ 软 `combined` 会松绑「不出最闲集合」；故意 `prefill_cost=None`、钉 V2。[17]

**③ 「调参从哪起？」**  
→ 先定 mode（追命中 vs 控长尾）；unified 从三系数 1.0 对齐翻盘量级；gated 调 topn。无 A/B 不报最优权重。
