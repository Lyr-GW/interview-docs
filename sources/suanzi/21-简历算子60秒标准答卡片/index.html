    21 · 简历×算子：8 题 × 60 秒标准答¶

    从 10 抽出高频题；每题：60 秒答 + 追问收口一句。
    练完对照 12、13。

    1. TTFT −70% 的算子解释？¶

    60 秒
    客户场景 4K、高前缀重复。我做 KV 亲和：token 级最长前缀匹配，把请求打到已有 cache 的实例。省的是 Prefill 算子链（PFA + 大 M Linear/FFN），Prefill 计算密集，所以 TTFT 掉得多。不是把 PFA kernel 加速了 70%。

    追问收口
    E2E −50% 更小，因为 Decode（IFA+FFN+sample）还在；见 13。

    2. 为什么必须 token 级匹配？¶

    60 秒
    字符级容易被 system prompt / tools 注入骗过，出现假前缀。假命中会复用错误 KV，Attention 数学输入错，或被迫回退满量 Prefill。token 级 + 全局索引保证「复用的 cache 与真实 token 前缀一致」。

    追问收口
    这是正确性约束带来的性能；见 14、kv knowledge/11。

    3. bitmask 在 NPU 上是什么？你写过算子吗？¶

    60 秒
    在 LM Head 之后、采样之前，对非法 vocab 位置置 −inf，属 Vector element-wise。我打通 Schema→xgrammar→Matcher→NPU 侧屏蔽；编译缓存是 SHA256 + FIFO≈100（命中不调序，勿说 LRU）；并修过异步错位。没有手写 AscendC bitmask kernel，是框架编排现有能力。

    追问收口
    相对 IFA/FFN 通常不是主瓶颈；先保证不错位。见 15。

    4. 异步 mask/采样错位怎么发现、怎么理解？¶

    60 秒
    约束解码与异步调度叠加时，可能用「过期合法集」去 mask「当前步 logits」。表现是偶发非法结构或空集。根因是 步进契约，不是 FA tiling。修复是对齐 slot/步号。

    追问收口
    和 aclgraph 的关系：mask/sample 常图外，更要显式同步。见 11 §1.2。

    5. unified vs load_gated？¶

    60 秒
    unified 更追前缀命中；load_gated 在亲和同时看实例负载，避免为了命中把请求打到过热 Decode 实例。过热时 IFA/FFN 争带宽，TPOT/吞吐变差，抵消 Prefill 收益。

    追问收口
    调度目标函数要同时看 Prefill 节省与 Decode 拥塞。见 07、09。

    6. PD 分离时 KV 传还是重算？¶

    60 秒
    本地命中最好。否则比较传输时间与重算 Prefill 时间：体积 ≈ S×L×D_kv，除以有效带宽得 T_xfer；和 T_prefill(S) 比。长前缀+够用带宽倾向传；短前缀或网络差倾向重算。MLA/KV 量化减小体积，传输更香。

    追问收口
    我支持双形态调度；传输协议细节不夸成个人交付。见 16。

    7. GE 和 aclgraph 你怎么选？和结构化输出什么关系？¶

    60 秒
    aclgraph 省 Host 下发；GE 省 Device 融合/内存。Host-bound 优先图捕获；Device-bound 要融合/量化。结构化输出的 mask/sample 动态多，常留图外，中间层可捕获。

    追问收口
    二者可叠加；已 Device-bound 只开 aclgraph 收益小。见主笔记。

    8. 没写过 AscendC，凭什么谈推理优化？¶

    60 秒
    我交付在 L5/L4：特性链路与调度，直接动 TTFT/正确性。算子层我补齐 Roofline、PFA/IFA/MLA 选型、GE/aclgraph、MoE/MC2 动机，能把调度对齐到算力/带宽，并与算子团队用同一套归因语言协作。不谎称手写生产 kernel。

    追问收口
    分层模型见 12 §5。

    练习用法¶

    • 计时 60 秒，录音；超过则删形容词。
    • 每题故意追问「所以你是算子优化？」——练收口到边界。
    • 次日只练 1/3/6/8 四题保底。
    • 开场前过一遍 24-答辩用词对照.md（FIFO / 少 Prefill / Mooncake 边界）。
