    块 1 · 投机（0:00–3:30）¶

    必背句（约 90s）¶

    Decode 小 batch 是 memory-bound。投机 = 便宜 draft 猜 k → target 一次前向并行验。
    账本：延迟≈(T_draft+T_verify)/τ；大 batch 变 compute-bound，投机常不划算。
    演进一句：Medusa 去独立模型 → EAGLE 特征/动态树 → MTP 预训练联合 → DFlash 并行块 → DSpark 半自回归+按负载裁 verify。
    MindIE：MTP / Lookahead / Memory 三插件；verify 多走贪心比对，≠ vLLM 默认概率拒绝采样。
    红线：MTP×结构化 = Serving 入口硬互斥；无自测曲线**不报**加速比。

    顺序 卡 只背什么

    自测 1 问¶

    问：面试官说「上周 DeepSeek 新投机论文」你怎么开？
    答：DSpark = Confidence-Scheduled + Semi-AR；相对 MTP 修 suffix/负载；公开加速标出处非我实测 → 翻 61。

    ⏱ 验收：闭卷 90s 讲完「账本 + 箭头 + 三插件 + 互斥」不卡壳。

    块 2 · vLLM 配置（3:30–7:00）¶

    必背句（约 90s）¶

    我不背配置名单，按**目标**选型：
    TTFT → enable_prefix_caching（块哈希复用）+ 视场景 enable_chunked_prefill（长 P 与 D 混批；TTFT 可能换稳 TPOT）。
    吞吐/稳 TPOT → max_num_batched_tokens / max_num_seqs 双预算 + cudagraph_mode（消 launch；小 batch 收益大）。
    低并发延迟 → speculative_config；接受率掉或大 batch 慎开。
    装模/池 → TP、kv_cache_dtype/fp8、gpu_memory_utilization。
    一句定调：旋钮背后是 PagedAttention + Continuous Batching 的**预算与复用**，不是魔法开关。

    自测 1 问¶

    问：开了 chunked prefill，为何 TTFT 可能变差？
    答：prompt 多步才算完，换的是 decode 不饿死与更高利用率 → 09 #2 / 44。

    ⏱ 验收：任抽「降 TTFT / 稳 TPOT / 低并发」各给 2 个旋钮 + 一句原理。

    块 3 · Mooncake（7:00–11:00）¶

    必背句（约 75s）¶

    Mooncake 不是单体：
    Transfer Engine = 怎么搬（RDMA/TCP、拓扑选路）；
    Store = KV 放哪、何时逐（Master 元数据，数据面点对点）；
    Conductor = 谁有前缀 + 选路（订阅 kv-events；不负责搬）。
    Motor = 决策层：查 Conductor → 打分选实例；真正存/传走引擎侧 Connector→Store/TE。
    Connector ≠ Mooncake 官方组件，是 vLLM/SGLang 适配层。
    红线：D 实例**不注册** Conductor；−70% = **测算**不是论文/客户 log。

    必翻卡¶

    顺序 卡 只背什么

    2 12 / 25 Removed / 假阳≠RR
    3 64 · 66 有余量再翻

    自测 1 问¶

    问：为何只上 Store 不上 Conductor 不够？
    答：Store 不知全局路由；缺索引与代价选路 → 13。

    ⏱ 验收：60s 电梯稿不看稿；能答「Connector 是谁的」。

    块 4 · 跳槽 / 软性（11:00–14:30）¶

    必背句（约 60s）¶

    拉力 > 推力。拉力：深耕推理服务系统，去把加速当核心竞争力、有大规模部署的团队；我能带 0→1（结构化 + KV 亲和）。
    推力只讲**业务方向**：配套硬件 + 引擎拥抱 vLLM → 框架层深度空间预期收窄；势能最好时换赛道。
    不说：涨薪、嫌加班、部门大家都跳、对前东家文化抱怨。
    稳定性：方向对会沉下来；这次是锚定方向，不是跟风。
