    1 · 开场 3 分钟¶

    1.1 先过脑：红线 8 条（链 03）¶

    # 正确 禁止出口
    1 编译缓存 FIFO / 100 LRU / 128
    2 Tokenizer：本地 model_path 同源 运行时从引擎拉取
    3 bitmask：torch NPU 组合 masked_fill_ 自研 fused kernel
    4 xgrammar：字节级 PDA 仓内随口「FSM」当严谨定义
    5 MTP × 结构化：入口硬互斥 「可以一起开」
    6 KVA：D 不注册 Conductor（仅 P/U） D 也上报索引
    7 TTFT−70%：代表性测算 「客户 raw log 实测 −70%」
    8 guidance：线上仅 xgrammar；多后端=预留抽象 「我们线上有 guidance」

    30s 自检口诀：100 FIFO → 测算非日志 → torch 组合 → MTP 互斥 → D 不注册。

    1.2 自我介绍（定稿旁链 37）¶

    可背全文：37-三分钟自我介绍定稿——默认 90s；对方要「多讲讲」用 180s。下表仅留骨架；红线钉子以 03 / 37 §0 为准。

    版 何时用 三条线怎么压
    90s 默认开场 SO（FIFO/100 + torch 非 fused）→ 亲和（查索引 + 测算 70%）→ Tool（协议/流式；Server 半句）
    180s 「三分钟 / 展开」 上列 + guidance 预留、MTP 互斥、D 不注册、通信非主战场、最熟=亲和+SO

    骨架口播（忘词急救 · 非定稿）：

    昇腾 MindIE 推理服务，三条线：结构化（PDA→bitmask，FIFO/100）→ Motor 亲和（tokenize→Conductor，TTFT−70% 是**测算**）→ Tool Call 协议/流式；不吹 kernel。可从简历往下挖。

    接话钩子：结构化→02/16/19；亲和/TTFT→38/12/17/18；Tool→28/ir/14；Server→14/35；Mooncake/HCCL→13/36；投机/配置→01/09；高压→39。

    完整禁踩雷对照见 37 §4（与下文 §5 禁语清单一致，定稿优先翻 37）。

    3 · 二面失分区急救（各链本夜卡）¶

    失分区（平安二面） 60s 开口锚点 翻卡 一句话止血
    投机解码 系统性失守 拒绝采样四步 + EAGLE→MTP→DSpark；「无损」=目标分布一致 01 B；深文 ir/02 大 batch 接受率掉；无曲线不报加速比；与 SO：硬互斥
    vLLM 配置 零输出 按目标：TTFT→prefix/chunked；吞吐→CG/预算/TP；正确性→勿乱开 async×SO 09；11 开口四层不漏；忌背十类名却讲不清旋钮
    Mooncake 只会用不懂底层 Conductor 调度 / Store 存 / TE 传；Motor=决策层查 Conductor；HCCL=集合≠点对点 KV 13；边界 36；04/07 Connector≠官方三层；通信非主战场；不报假加速比
    跳槽 答成涨薪&嫌加班 拉力（推理加速核心竞争力）> 推力（业务方向收窄）；势能最好时换赛道 08 禁加班/跟风/编司龄；压力下先认缺口再切 SO+亲和

    失分后 15s 救场模板（见 08 压力轮）：

    「这块我跟踪不够系统，记下来了。工程交付我有把握——结构化 0→1 或 Motor token 级亲和，您想先听哪块？」

    5 · 「绝对不要说」清单（10 条）¶

    # ❌ 禁止 ✅ 改口 锚
    1 编译缓存是 LRU / 128 FIFO / 100，命中不调序 03 #1
    2 TTFT−70% 是 客户 raw log / 实测 代表性测算（可白板公式） 03 #7；38；02
    3 我们线上有 guidance 唯一落地 xgrammar；guidance=预留抽象 03 #8；02 §1
    4 D 实例也注册 Conductor D 不注册；仅 P/U 03 #6；07
    5 bitmask 是 自研 fused / AscendC kernel torch 算子组合 masked_fill_；可优化点是 fused 03 #3；19
    6 编造 LOC / 「删了 1 万行」精确数 路径+模块拆解；数字 待 cloc 14；15 E4
    7 编造 投机/Spec/LMCache 加速比 只讲机制；无压测不报百分点 01 B；04；15 E2
    8 MTP 与结构化 可以一起开 / 已打通 入口硬互斥；插件未联调 03 #5；23；02 §7
    9 跳槽因 涨薪 / 嫌加班 / 跟风 拉力>推力；方向收窄；势能最好时换 08
    10 「我最擅长做得不多」或 谎称生产 kernel/HCCL 点 SO+亲和；算子/通信=协作归因；HCCL 边界见 36 08；06；36

    附加禁语：把现状树行数冒充重构删行；把 Tool Call 行数混进 5000+；把 Connector 说成 Mooncake 官方三层之一。

    6 · 应急：被问到没写过的题¶

    完整 12 场景话术库（不会 / 没测过 / 数字对不上 / 代码抓包 / 竞品没写过 / 问穿了 / 要客户日志 / 追 LOC / 追加速比 …）：见 39-高压追问应急话术卡。
    下表只留三步框架 + 3 个短例；上场高压优先翻 39。

    框架三步（边界诚实 → 迁移已知 → 反问场景）：

    1. 边界诚实（5–10s）

      「这个具体题/论文我没在生产里落地过，我不瞎编实现细节。」

    2. 迁移已知（30–45s）
      从本夜/简历主场抽 同构问题：

    3. 调度/缓存类 → Motor precise vs 近似树（12/17）或 Dynamo cost（01 A）
    4. 约束/正确性 → xgrammar bitmask + async 错位（16/19）
    5. 传输/PD → 选型 24 + handoff 07 + Mooncake 13 + HCCL 边界 36；ZMQ 通知面 25

    6. 性能归因 → Profiling 四层（01 D）；TTFT 五段/测算 → 38
      讲清：问题形态、取舍轴、失败模式，不假装读过对方仓库。

    7. 反问场景（10s）

      「你们更关心延迟、吞吐，还是正确性/多租户？我按那个目标对齐一版设计权衡。」

    短例（更多见 39）

    被问 诚实 迁移 反问
    某新投机论文细节 没跟到实现 draft-verify + 接受率掉点（01 B） 低并发加速还是大 batch 稳定？
    某 KV 池化产品名 没部署过 Conductor 真索引 vs 本地树（12/13） 要精确命中还是近似省中心？
    某算子融合名 没写过该 kernel Roofline + FA/CG 边界（06） 算力墙还是带宽墙？
    客户日志 / −70% 无 raw log 测算白板 38 复测要分位还是机制？

    禁止：用「应该差不多」编 API；用假 LOC/假加速比填沉默。
