    61 · DSpark 一页口述卡（可背）¶

    本夜续批（2026-07-15 · 04:29）

    边界钉死：论文/公开数跟读；**非**本仓生产实测。无曲线不把公开%说成「我线上测过」。

    1 · 60 秒电梯稿（可直接背）¶

    DSpark（2026-06-27，DeepSeek；随 DeepSpec 开源）= Confidence-Scheduled（按 GPU 实时负载裁送验草稿长度 k）+ Semi-Autoregressive（并行主干一次出块 + 轻量低秩 Markov 头从左到右偏置校正，补块内连贯）。[文档·ir/02]

    相对 MTP 的卖点一句：MTP 草稿头仍串行小前向；DSpark 用并行压 T_draft，用 Markov 抬 τ、用负载调度兜住高并发下 T_verify 暗涨——专修「串行 draft 贵 + 纯并行 suffix decay + 大 batch 验证不免费」。[文档·34/ir/02]

    诚实边界：我这边是**论文跟读 + 演进线对齐**，**没有**本仓/客户线上压测曲线；公开加速比只能标出处复述，不能说成我测的。[文档·02 诚实约定]

    金句：半自回归修连贯；置信度按负载裁 k；相对 MTP 修串行 draft；数字=公开非实测。

    2 · 两词拆开（20s）¶

    词 一句
    Semi-Autoregressive 主干并行（快）+ Markov 薄层串行校对（连贯）；串行的是偏置，不是再跑一层 transformer。[文档·ir/02]
    Confidence-Scheduled confidence 估存活概率 → 负载低多验抬 τ，负载高少验防 T_verify 抢算力。[文档·ir/02]

    vs DFlash 半句：DFlash 纯并行易 suffix decay；DSpark = 类并行主干 + Markov + 负载调度。[文档·34]

    4 · 快问 6（10–20s / 题）¶

    # 问 答要点
    1 DSpark 全称/两词？ Confidence-Scheduled + Semi-Autoregressive。[文档]
    2 相对 MTP 卖点？ 并行压 draft 延迟 + 负载裁 verify；MTP 仍串行头。[文档]
    3 相对 DFlash？ Markov 补连贯 + 按负载裁 k；修 suffix decay / 高并发。[文档]
    4 加速多少？ 公开：vs MTP-1 Flash +60–85% / Pro +57–78%；非我实测。[公开·ir/02]
    5 你们线上开了吗？ 跟读论文与演进；未做本仓生产压测，不报自家%。[诚实]
    6 无损吗？ 文档称输出无损（走拒绝采样族）；大 batch 仍可能负收益吞吐。[文档/机制]

    5 · 追问 3 连¶

    ① 「57–85% 是你测的吗？」
    → 否。复述自专题已收录的 DeepSeek-V4 **公开线上**相对 MTP-1 口径；我这边无同配置复现曲线。要谈落地，先讲 (T_draft+T_verify)/τ 与接受率失效，再谈是否值得接 DSpark 类 method。[文档·ir/02 + 诚实]

    ② 「Markov 头为何几乎不涨 T_draft？」
    → 主干一次出 base_logits；头是 V×r/r×V 低秩偏置，每步≈embedding + 小 GEMM，不是再跑 transformer；可进同一 CUDA Graph。rank 以配置为准，未查权重勿报死数。[文档·01B/ir/02]

    ③ 「为何还要发明 DSpark（上周新论文）？」
    → DFlash 修了串行 draft 延迟，留下块内不连贯 + 高并发 verify 暗涨；DSpark 用半自回归修 τ、用负载调度兜分子——专修上一代劣势场景。[文档·34]

    6 · 易混两刀¶

    易混 正确口径
    DSpark = DFlash DFlash 纯并行；DSpark = 并行主干 + Markov + 负载调度
    公开 +60–85% = 我司收益 公开线上 vs MTP-1；本仓未复现则只讲机制

    与 MindIE 关系一句：MindIE 现网三插件是 MTP/LA/Memory，**不是**已产品化的「DSpark 插件」——点名新论文时走演进线，勿说「我们已上 DSpark」。[文档·34]

    7 · 一页抄写版¶

    DSpark = Confidence-Scheduled + Semi-Autoregressive
  并行主干 + 低秩 Markov 校对 + 按 GPU 负载裁 verify 长度 k

相对 MTP: 修串行 draft；并行+调度

             τ: vs EAGLE-3 +26–31% / vs DFlash +16–18%
