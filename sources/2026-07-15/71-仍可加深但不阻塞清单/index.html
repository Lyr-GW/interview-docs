    0 · 怎么用（20 秒）¶

    场景 动作
    睡前/通宵还想写 停；本页主题一律不当本夜必做
    面试被点名 用本页「怎么挡」+ 链卡；勿编生产数字
    面试后有空 按兴趣加深；仍勿伪造 PR/加速比

    口令：边界诚实 → 同构迁移 → 反问场景（母本 39）。

    1 · 主题清单（有空再补）¶

    1 · 长上下文（128K+ / YaRN / 窗口外推）¶
    现状 有 KV 量化/slot、传值阈值（64）、chunked（44）旁链；无独立「长上下文系统」口述卡
    加深方向 RoPE/YaRN 一句；paged KV 显存粗算；长 prefill 与 PD/chunked 选型
    若被问到怎么挡 「长上下文我主战场是 KV 显存与搬移成本，不是训侧外推论文；slot/量化/传不传按带宽阈值拆——外推细节我没生产落地，按你们窗口长度我可以对齐一版预算。」→ 翻 29 KV FP8 · 64 · 44 · 高压 39

    2 · Multi-LoRA / 多适配器服务¶
    现状 SGLang Radix 有 LoRA 命名空间旁注（48/sglang/12）；简历无 Multi-LoRA 交付
    加深方向 adapter 热加载、显存池、与 prefix 命名空间隔离
    若被问到怎么挡 「Multi-LoRA 服务化我没在现网做 owner；同构问题是 多租户权重切片 + 缓存命名空间——我对 prefix/亲和隔离熟，LoRA 调度细参我不编。」→ 48 · 27 · 39

    3 · Embedding Serving（向量/检索侧推理）¶
    现状 主线是 LLM decode/PD/亲和；无 embedding 专卡
    加深方向 batch 动态、短序列吞吐、与 LLM 共享集群的队列隔离
    若被问到怎么挡 「Embedding serving 不是我简历主交付；推理服务共性仍是 批调度与资源隔离——LLM 侧我有 Motor/结构化证据，embedding 具体栈我按你们组件对齐，不瞎报 QPS。」→ 39 · 11

    4 · NIXL 深挖（协议/部署/调参）¶
    现状 边界已有：04 Part B；与 HCCL/TE 正交见 36
    加深方向 NixlConnector 路径、G4 运维、与 Mooncake TE 对照实验
    若被问到怎么挡 「NIXL 我当 数据面抽象（Dynamo G4 / llm-d PD 常用）；我主场是 Conductor 索引 + Mooncake 栈，NIXL 未生产深操——不报自制 GB/s，只讲层：传≠索引。」→ 04 · 36 · 26 · 01A

    5 · 开源贡献实物（博客 / upstream PR / RFC）¶
