    32 · 拒绝采样白板特训卡（可背）¶

    本夜续批（2026-07-15 · 02:48）
    用途：把 01 §B 拒绝采样四步拆成**可默写白板**——草稿生成 → target 验证 → 接受/拒绝 → 期望加速比直觉；钉死**无损定义**与**大 batch 失效一句**。
    深文：interview-review/02。
    旁链：演进 3 分钟默背见 34；MTP×SO 见 23；本卡**只练白板核**，不重背 EAGLE→DSpark。

    交叉引用¶

    文档 用途
    01 §B 60s 电梯稿、演进箭头、快问母本
    interview-review/02 draft-verify、失效场景完整版、公式推导
    易混 「无损」=永远加速；Medusa typical ≠ 严格拒绝采样；MindIE 贪心 verify ≠ vLLM 概率拒绝采样

    数字标注：[文档已有] / [机制推导]。无压测曲线不报具体加速比。

    1 · 60 秒电梯稿（开口定调）¶

    Decode 小 batch 是 memory-bound：权重搬运贵、算力闲。投机 = 便宜 draft 猜 k 个 → target 一次前向并行验。[机制·02]

    验证用**拒绝采样**：α=min(1, p/q)；拒则从 norm(max(0,p−q)) 重采样——输出分布 ≡ 纯 target，叫**数学无损**。[机制·02]

    账本：单 token 延迟 (T_draft+T_verify)/τ；要它 < T_target 才划算。大 batch 变 compute-bound → verify 不再免费，吞吐常掉。[机制·02]

    2 · 完整白板步骤（可默写）¶

    白板从左到右四格；边写边说，约 90–120 秒。

    步① · 草稿生成（Draft）¶

    上下文 c 已知
draft 模型（小模型 / 草稿头 / MTP…）自回归或并行块
产出：x1, x2, …, xk
顺带记下各步草稿分布 q_i（或等价 logit）

    口述一句：draft 便宜、可错；目标是抬高「一次 verify 覆盖的期望长度」。[文档·02]

    步② · 目标模型验证（Target Verify）¶

    target 对「c + x1..xk」做一次前向
得到各位置真实分布 p1, p2, …, pk（及可选 bonus 位）
关键：k 个位置的 p 在同一次前向里并行拿到
（相对「k 次自回归 target 步」）

    口述一句：投机的算力账本发生在 verify 一次换 k 步候选。[机制·02]

    步③ · 接受 / 拒绝（逐位）¶

    从 i=1 起：
  α_i = min(1, p_i(x_i) / q_i(x_i))
  以 α_i 概率接受 x_i
  若拒绝：
    · 丢弃 x_i 及之后全部草稿
    · 从修正分布 r = norm(max(0, p_i − q_i)) 采样 1 个 token
    · 本轮 verify 结束（后面草稿作废）
若 1..k 全接受：
  · 还可 bonus：从 target 在下一位置的分布再采 1 个（可选实现）

    金句：拒绝不是「随便换个词」，而是按 standard 拒绝采样 修正，保证分布对。[机制·02]

    步④ · 期望加速比直觉（公式一页）¶

    τ  = 本轮期望接受长度（含 bonus 则按实现计）
T  = T_draft + T_verify     # 一轮 draft→verify 墙钟

单 token 延迟 ≈ T / τ
要对齐自回归：需  T/τ  <  T_target

加速比直觉 ≈  T_target · τ  /  (T_draft + T_verify)  > 1

失效三叉：
  τ↓（猜不准 / 高温）
  T_verify↑（大 batch · compute-bound）
  T_draft↑（串行 draft 太贵）

    3 · 无损定义（必背一句 + 边界）¶

    项 口径
    是什么 最终 token 序列的分布与 只用 target 自回归 一致（lossless）。[机制·02]
    靠什么 standard 拒绝采样：min(1,p/q) + 拒后 norm(max(0,p−q))。[机制·02]
    不是什么 不是「一定更快 / 一定更高吞吐」；慢了也仍叫无损。[机制·02]
    易混 Medusa 等 typical acceptance 路径常 非严格无损——开口先问验证器是不是拒绝采样。[文档·02]
    引擎差 MindIE MTP 常见 贪心逐位比对；≠ vLLM 默认概率拒绝采样——别混称同一套。[文档·01B]

    4 · 大 batch 失效（一句钉死）¶

    一句：小 batch 时 verify 吃的是 memory-bound 闲置算力；大 batch / GPU 已饱和后 decode 变 compute-bound，并行验 k 个草稿是从别的请求嘴里抢算力 → 单请求或略快，系统吞吐常下降。[文档·02]

    补刀：高负载常缩短 k / 关投机；无曲线只谈定性，不报假百分点。[文档·02]

    5 · 默写检查清单¶

    闭卷默写；勾满过关。

    四步骨架

    • [ ] draft → x1..xk + q
    • [ ] target 一次前向 → 各位 p
    • [ ] α=min(1,p/q)；拒 → 丢弃后缀 + norm(max(0,p−q))
    • [ ] 全接受可写 bonus（建议）

    公式与失效

    • [ ] 延迟≈(T_draft+T_verify)/τ + 加速比直觉式
    • [ ] 失效三叉：τ↓ / T_verify↑ / T_draft↑
    • [ ] 无损 =「分布≡target」，划掉「永远加速」
    • [ ] 大 batch 一句：compute-bound → verify 抢算力 → 吞吐↓

    红线（口头）

    • [ ] MindIE 贪心 ≠ vLLM 概率拒绝采样；不编加速比；typical ≠ 严格无损

    6 · 快问 8（10–20s / 题）¶

    1. 四步顺序？ → draft 出 x,q → target 一次前向得 p → α 接受/拒后修正采样 → 可选 bonus。[01B/02]
    2. α 公式？ → min(1, p(x)/q(x))。[机制]
    3. 拒绝后采样哪个分布？ → norm(max(0, p−q))；并丢弃该位之后草稿。[机制]
    4. 无损指什么？ → 输出分布 ≡ 纯 target；**非**吞吐保证。[机制]
    5. 加速比直觉？ → T_target·τ/(T_draft+T_verify)>1；单 token 延迟 T/τ。[机制]
    6. 大 batch 为何失效？ → compute-bound，verify 抢共享算力，吞吐常↓。[文档·02]
    7. τ 是什么？ → 每轮期望接受长度（接受率越高通常越大）。[机制]
    8. 为何 memory-bound 才划算？ → 闲置算力「白捡」做并行 verify；饱和后不再免费。[机制·02]

    7 · 一页抄写版（与默写对齐）¶

    Draft:  c → q, x1..xk
Verify: target(c+x) → p1..pk     # 一次前向
Accept: α=min(1,p/q)；拒→norm(max(0,p−q))+截断后缀
Bonus:  全接受可再采 1 个（target）
无损:   分布≡target自回归；≠永远加速
账本:   延迟=(Td+Tv)/τ ；加速≈Tt·τ/(Td+Tv)
大batch: compute-bound → Tv 抢算力 → 吞吐↓

    高温追问：E[α]↓→τ↓，无曲线只谈定性。bonus 从 target 采，不破无损。[机制·01B]
