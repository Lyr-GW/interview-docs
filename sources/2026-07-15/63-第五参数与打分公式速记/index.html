    1 · 60 秒电梯稿（可直接背）¶

    KV 亲和打分五旋钮：mode 选软融合还是硬门控；其余四系数只在对应语义里生效。默认 unified，三权重全 1.0，load_gate_topn 字面 0→语义 2。[配置默认·17]

    unified（默写）：
    score = prefill_load_scale × max(0, isl − overlap_credit × matched) + load_weight × load，取 min。量纲全是 token；命中够大可翻盘负载。[机制·ir/12]

    load_gated（默写）：① 负载升序取 Top-N ② N 内 matched 降序（平局更低负载）。亲和**出不了**最闲集合——有限 load_weight 表达不了这硬上界，故必须单开模式。[17]

    金句：软和可翻盘；硬门控不出 Top-N；默认 unified / 1.0 / topn→2。

    2 · 五参数速记卡（物理含义 + 调大/调小 + 默认）¶

    参数（配置键） 物理含义（一句） 调大直觉 调小直觉 默认 标注
    kv_affinity_mode 选 unified（软加权和）或 load_gated（字典序硬门控） —（枚举） — "unified" [配置默认]
    kv_affinity_load_weight load 项权重；1.0 = 1 token 排队 ≡ 1 token 待算 prefill 更怕热点，空闲机更容易赢 更追命中；0 = 纯亲和 1.0 [配置默认]
    kv_affinity_overlap_credit 命中 1 token 折抵多少 prefill 工作量 命中更「值钱」，更易翻盘 命中折扣缩水，更像无缓存 1.0（命中即全额省） [配置默认]
    kv_affinity_prefill_load_scale 整项 prefill_cost（待算量）的缩放 放大「还要算多少」的权重 缩小待算项，负载项相对更响 1.0 [配置默认]
    kv_affinity_load_gate_topn 仅 load_gated：最闲集合宽度 门控变宽，亲和可选面更大 门控变窄；1 ≈ 纯 LB 字面 0 → 运行时 2 [配置默认·代码哨兵]

    旁路常量（常一起默）：

    项 值 标注
    _AFFINITY_CANDIDATE_TOPK 3 [代码事实] V2 候选宽
    负载分 active_tokens + 0.3×active_kv_cache 0.3 启发式，非第一性 [17]
    Conductor 查询超时 0.2s → 回退 LB [配置] 见 62

    3 · 公式骨架（可默写白板）¶

    3.1 unified（默认 · 软融合）¶

    prefill_cost = max(0, isl − overlap_credit × matched)
score        = prefill_load_scale × prefill_cost + load_weight × load
选 endpoint:  argmin(score)   // 平局偏低 prefill_cost（更好亲和）
    性质 一句
    量纲 两项都是 token
    翻盘 能：命中大 → prefill_cost 小 → 可压过高 load
    Herding unified 升 V3 全量全局重排（#304）
    退化 load_weight=0 → 纯最长前缀

    3.2 load_gated（硬门控）¶

    ① candidates = nsmallest(N=load_gate_topn, endpoints by load)
② 在 candidates 内: matched 降序；平局取更低 load
禁止: 亲和把请求拉出「最闲 Top-N」
    性质 一句
    亲和 vs 负载 亲和**永远不能**压过门控
    Herding 钉 V2 top-k；故意不带 prefill_cost 软重排
    退化 topn=1 → 近似纯 LB
    为何单开模式 字典序硬约束 ≠ 任何有限加权和

    口述禁语：

    「把 load_weight 调很大就能冒充 load_gated」——错；weight→∞ 会压死亲和变纯 LB，仍不是「先负载再亲和」的字典序。

    4 · 翻盘临界（测算例 · 可算账）¶

    锚：isl=8192、matched=6000、三系数全 1.0。[测算·17/13]

    epA（高命中）: score = (8192−6000) + load_A = 2192 + load_A
epB（零命中）: score = 8192 + load_B
epB 胜 ⟺ load_A − load_B > 6000   (= overlap_credit × matched)

    调参直觉：↑credit → 空闲更难翻盘；↑load_weight → 空闲更容易赢；↑scale → 命中优势更响。均为**机制账**，非压测曲线。

    选型：unified = 追命中可翻盘；load_gated = 控长尾护 Decode。打分再精，假阳仍靠 ZMQ Removed（12）；utilization/speculative 未进分。

    5 · 快问 8（10–20s / 题）¶

    1. 五参数名？ → mode / load_weight / overlap_credit / prefill_load_scale / load_gate_topn
    2. 默认？ → unified；三系数 1.0；topn 字面 0→语义 2。[配置]
    3. unified 公式？ → scale×max(0,isl−credit×matched)+w×load，取 min
    4. load_gated 两步？ → 负载 Top-N → N 内最长 matched
    5. load_weight=0？ → 纯亲和
    6. topn=1？ → ≈纯 LB
    7. 为何不能用 weight 冒充 gated？ → 硬字典序 ≠ 有限加权和
    8. 翻盘例 matched=6000？ → 空闲要赢须 Δload >6000；测算

    6 · 追问 3 连¶

    ① 「和 Dynamo overlap 一回事吗？」
    → 直觉同类；Motor 单层 credit×matched，Dynamo 还有 Device/Host/Disk/Shared 分层。先默 Motor，点名再展开（01 A）。

    ② 「为何 gated 不升 V3？」
    → 软 combined 会松绑「不出最闲集合」；故意 prefill_cost=None、钉 V2。[17]

    ③ 「调参从哪起？」
    → 先定 mode（追命中 vs 控长尾）；unified 从三系数 1.0 对齐翻盘量级；gated 调 topn。无 A/B 不报最优权重。

    7 · 一页抄写版¶

    mode: unified|load_gated  默认 unified
w=1.0 ↑怕热点  0=纯亲和 | credit=1.0 ↑命中值钱
scale=1.0 ↑待算项 | topn: 0→2  (1≈纯LB)

unified: score=scale*max(0,isl-credit*matched)+w*load ; argmin; 能翻盘; V3
load_gated: Top-N by load → max matched ; 不出集合; 钉 V2
翻盘(测算): matched=6000 → Δload>6000
