    42 · 45 分钟模拟面试串联剧本（面试官视角）¶

    本夜续批（2026-07-15 · 03:33）
    用途：严格按**面试官**出题节奏，把 45 分钟压成六段时间盒；每段只测一件事。
    考生用法：找人对练或自问自答；盖住「预期翻卡 / 通过标准」先答，再对照。
    导航母本：20。开场定稿 37；高压模板 39。
    口径：与 03 冲突时以 03 为准（FIFO/100；TTFT−70%=测算）。

    0 · 时间盒总表（钉死 45′）¶

    # 段 时限 主测能力 主翻卡
    1 自我介绍 0–3′ 开口嵌红线、交棒 37（旁链 20§1）
    2 结构化深挖 3–13′ SO 链路 + 第三层诚实 01旁链/02/16/19/23
    3 KV 亲和 13–25′ Motor + 假命中/Herding/测算 12/17/18/25/38
    4 投机白板 25–33′ 拒绝采样 + 演进一句 32 + 01 B / 34
    5 vLLM 配置 33–40′ 按目标选配置 09（11 加深）
    6 软性收尾 40–45′ 跳槽/压力轮 08 + 39

    面试官总原则：前 3′ 听钩子；中段宁可深挖一条也不扫十题；末段故意高压一次，看改口能力。
    考生禁语扫一眼：20 §5 / 03 八条。

    1 · 自我介绍（0–3′）¶

    项 内容
    时限 3′（考生默认 90s；对方说「多讲讲」才放 180s）
    主问 「先介绍一下你自己，以及你最近在做的推理相关工作。」
    预期翻卡 37；骨架急救 20 §1.2
    通过标准 ① 三条线可辨：SO / KV 亲和 / Tool（或 Server 半句）② 开口嵌 FIFO/100 与 测算（或等价红线）③ 不吹自研 fused / 客户 raw log ④ 结尾交棒「可从简历往下挖」
    故意高压追问 ×1 「你简历写 TTFT 降 70%，是客户线上实测吧？」→ 必须立刻改口「代表性测算」；模板见 39 场景「要客户日志」

    面试官听点：钩子是否清晰（下一刀砍哪条）；数字是否可辩护。
    考生急救：忘词只背 37 90s；禁 LRU/128、禁「实测 −70%」。

    2 · 结构化输出深挖（3–13′）¶

    项 内容
    时限 10′（前 4′ 链路；中 4′ 工程坑；后 2′ 互斥/边界）
    主问 「结构化输出你们怎么做的？从 Schema 到采样讲清楚；编译缓存策略是什么？」
    预期翻卡 链路/02§1；缓存红线 03；async 16；bitmask 19；MTP×SO 23；总览旁链 01
    通过标准 ① Schema→EBNF→字节级 PDA→bitmask→采样 ② 缓存 FIFO / 默认 100，命中不调序 ③ bitmask=torch 组合，非自研 kernel ④ 被问 MTP：入口**硬互斥** ⑤ guidance=预留，线上仅 xgrammar
    故意高压追问 ×1 「你们文档/旧口述写过 LRU 128，你现在说 FIFO 100，到底哪个真？」→ 立刻钉代码真相 FIFO/100；承认旧口径过时；见 39「简历数字对不上」

    面试官可加刀（可选，超时则砍）：async mask 错位三因；5000+ 别混 Tool/Server 行数。
    考生急救：被抓 LRU → 不辩解「差不多」，直接订正 + 源码层策略一句。

    3 · KV 亲和（13–25′）¶

    项 内容
    时限 12′（机制 5′ → 假命中/事件 4′ → 测算/Herding 3′）
    主问 「Motor KV 亲和相对 SGLang/vLLM Router 本地树，核心差在哪？假命中怎么防？」
    预期翻卡 假命中 12；Herding 17；Tokenizer 18；ZMQ 25；TTFT 38；Mooncake 三层 13
    通过标准 ① tokenize 前置 + Conductor 精确最长前缀（查索引 vs 猜缓存）② 假阳 vs 假阴取舍；ZMQ Removed 驱逐感知 ③ D 不注册 Conductor ④ −70% 能白板假设或标「测算」⑤ Herding：overlay→top-k→unified 演进一句
    故意高压追问 ×1 「Conductor 挂了 / 超时你怎么办？你们 P99 多少？」→ 讲 0.2s 超时+回退；无 P99 不报数（待补见 15）；高压模板 39「没测过」

    面试官听点：通知面（ZMQ）≠ 数据面（TE/Store）；勿把 Connector 说成 Mooncake 官方三层。
    考生急救：被逼客户日志 → 翻 38 测算假设清单，不甩 raw log。

    4 · 投机解码白板（25–33′）¶

    项 内容
    时限 8′（白板 5′ + 演进/失效 3′）
    主问 「白板画一下投机解码的拒绝采样；什么叫无损？大 batch 为什么容易掉？」
    预期翻卡 白板核 32；口述母本 01 B；演进 34；深文 ir/02
    通过标准 ① draft→verify→accept/reject 四步**可画 ② 「无损」=与目标分布一致（非「一定更快」）③ 大 batch / 低接受率失效一句 ④ 与结构化：**硬互斥（若追问）⑤ 无压测曲线**不报加速比**
    故意高压追问 ×1 「DSpark / 上周那篇你们线上开了吗？加速比多少？」→ 诚实：跟论文/演进线，未生产压测不报百分点；迁移 draft-verify + 接受率；见 39「追加速比」+ 01 B

    面试官听点：会不会只背名词不会画；会不会用假加速比填沉默。
    考生急救：白板卡壳 → 先写四步框再填细节；演进用 34 每代一句即可。

    5 · vLLM 配置串讲（33–40′）¶

    项 内容
    时限 7′（按目标 60s + 追问 2–3 旋钮）
    主问 「如果目标是降 TTFT / 提吞吐 / 保正确性，你分别会动哪些 vLLM（或同构）配置？为什么？」
    预期翻卡 09；Scheduler/CB 11；深文 ir/05
    通过标准 ① 按目标选配置，不是背十类清单 ② TTFT：prefix / chunked 等与 Prefill 相关 ③ 吞吐：CG / 预算 / TP 等一句原理 ④ 正确性：async×SO 等「勿乱开」意识 ⑤ 至少能讲清 2 个旋钮背后机制
    故意高压追问 ×1 「enable_prefix_caching 和你们 Motor 亲和是一回事吗？」→ 必须拆两层：引擎 PrefixCache/APC（27）vs 跨实例 查 Conductor；正交，不混谈

    面试官听点：是否只会念配置名；与 Scheduler 双预算能否挂钩。
    考生急救：不会某具体 flag → 讲目标轴 + 同构旋钮，不编 API（39「不会」）。

    6 · 软性收尾（40–45′）¶

    项 内容
    时限 5′（动机 3′ + 压力/反问 2′）
    主问 「为什么考虑看新机会？你最擅长和最薄弱分别是什么？」
    预期翻卡 08；压力轮旁链 39；「最擅长」对齐 37
    通过标准 ① 拉力 > 推力（推理加速核心竞争力 / 赛道势能）② 禁：涨薪、嫌加班、跟风、编司龄 ③ 最擅长点 SO+亲和；薄弱承认系统跟踪/算子协作，不自我贬低砸主场 ④ 被压能「先认缺口 → 切交付」
    故意高压追问 ×1 「就一年就想跳？是不是做不下去 / 被优化？」→ 势能最好时换赛道；方向收窄非能力否定；模板见 08 压力轮 + 39

    面试官听点：情绪是否稳；会不会又滑回涨薪叙事。
    考生急救：15s 救场句见 20 §3 / 08。
