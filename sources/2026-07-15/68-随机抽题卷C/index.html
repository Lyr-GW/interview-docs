    68 · 随机抽题卷 C（抽签默背）¶

    本夜续批（2026-07-15 · 04:54）
    用途：与 56 / 57 尽量不重复 的第三套抽签卷；只给题面，文末仅翻卡号。
    主题偏：传输阈值、Store、Connector、超时回退、五参数、MTP/DSpark、中场摘要交叉。
    构成：简历绑定 5 + JD 盲区 5 + 白板 3 + 软性 2 = 15 题。
    用法：同 A/B；建议 A→B→C 隔轮，勿同夜连刷三套满题。

    0 · 抽签规则（30 秒）¶

    1. 随机 5–8 题；全刷约 25–35′。
    2. 与 A/B 撞感 → 换刀口（阈值/Store/Connector/超时/五旋钮/DSpark），勿背旧稿。
    3. 盖住文末；对完纠偏只翻卡号对应正文。
    4. 数字必标性质：论文阈值 / 配置 / 目标·待补 / 公开非实测。
    区 题号 与 A/B 的刻意错位
    R R1–R5 超时回退/五参数/传值/MTP×SO/DSpark（非 FIFO/−70%/假命中主轴）
    J J1–J5 Store 驱逐/Connector 四维/传值阈值/Conductor 成本/中场仍薄
    W W1–W3 Eq 阈值/五旋钮默写/超时回退图（非拒绝采样+TTFT 五段主轴）
    S S1–S2 「仍薄」诚实 / 中场入口纪律（非跳槽主句+优缺点）

    1 · 题面（盖住文末再答）¶

    R · 简历绑定（5）¶

    R1. Conductor 查询超时配多少？超时/挂/无数据后回退到什么？LoadBalance 是不是轮询？P50<5ms / P99<20ms 是实测还是目标？

    R2. KV 亲和五旋钮：mode / load_weight / overlap_credit / prefill_load_scale / load_gate_topn 各一句物理含义；默认 mode 与 topn 字面 0 的运行时语义？

    R3. 「冷 Prefill 还能不能抓别处 KV？」能靠哪一层？重算 vs 传谁随前缀更亏？LLaMA3-70B、p=8192 的 A800/H800 带宽阈值各约多少、出处？

    R4. MTP × 结构化输出：入口硬互斥还是可一起开？产品理由与工程理由各半句；MindIE 三插件你能背哪三个名字？

    R5. 面试官点名 DSpark：两词拆开各一句；相对 MTP 卖点一句；公开加速比怎么说才不踩「我线上测过」？

    J · JD 盲区（5）¶

    J1. Mooncake Store：Master / Client 谁记账谁搬数？near-LRU 与 soft pin / hard pin / busy 驱逐优先级一句？Conductor「删冷 KV」这句话错在哪？

    J2. vLLM vs SGLang 接 Mooncake：PD 传输粒度、握手、跨实例前缀、路由 cache-aware——各一句谁更深？Connector 是不是官方三层之一？

    J3. Motor 打分有没有显式 Ttransfer？Conductor Algorithm 1 的 TTFT 直觉含哪三项？「有命中就拉」的 StoreConnector 缺哪一层账？

    J4. SGLang Gateway CacheAware 是否等于查 Mooncake 元数据？与 Motor→Conductor「猜 vs 查」怎么一句划界？

    J5. 对照中场摘要：本夜文档侧门禁过了吗？唯一开放环是什么卡号？明早强制入口三件套是哪三张？

    W · 白板（3）¶

    W1. 默写传值直觉骨架：重算 ∝ p²、传 ∝ p；标出 6 / 19 GB/s 对应哪两卡与性质（论文阈值）。

    W2. 默写 unified 打分式 + load_gated 两步（Top-N 再 matched）；旁注五参数默认值。

    W3. 画超时回退链：触发条件 → 亲和 None → LoadBalance →（再失败）round_robin；标 200ms 性质=配置。

    S · 软性（2）¶

    S1. 「你们线上 Conductor P99 / 投机加速比测过吗？」用 测算·待补·未压测 三词结构 45s 答；禁止编分位或百分点。

    S2. 「通宵准备了这么多，明早从哪开始？」60s 答强制入口 + 刷卷策略（可提 A/B/C 隔轮）+ 仍薄只跟 15；禁止说「我从头读完 interview-review」。
