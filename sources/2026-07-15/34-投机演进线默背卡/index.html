    1 · 总纲金句（开口 10s）¶

    演进是两条腿：把 draft 更准（多头→特征自回归→动态树→联合训练），把 draft/verify 更省（并行出块→半自回归校对→按负载裁 k）。每一步专修上一代的短板。[文档·02]

    账本一句（挂靠即可，详背 32）：延迟≈(T_draft+T_verify)/τ。[机制]

    2 · 每代一句「解决了什么」（倒背表）¶

    代 解决了什么（一句） 遗留短板（半句）
    Medusa 去掉独立 draft 模型，target 顶层加多头，零额外部署 头间独立、无块内依赖；typical 常非严格无损
    EAGLE-1 自回归挪到**特征层** + 真实 token 反馈，复用 emb/LM head draft 仍串行，T_draft∝k
    EAGLE-2 置信度≈接受率 → 动态长草稿树，免重训抬 τ 依赖置信度校准；本质仍串行
    EAGLE-3 去 l_fea 枷锁 + 直接预测 token + 多层特征融合，打破 scaling 训练更重；draft 仍自回归
    MTP **预训练联合**草稿头，推理白捡，对齐天然好 依赖模型自带权重；层间仍串行
    DFlash block diffusion 并行去噪，一次前向出整块，T_draft≈与 k 解耦 纯并行 → suffix decay
    DSpark 并行主干 + 低秩 Markov 补连贯 + 按 GPU 负载裁 verify 长度 工程最重；第三方落地仍在扩

    箭头串读（30s）

    Vanilla 证明可行 → Medusa 去独立模型 → EAGLE-1 特征层 → EAGLE-2 动态树 → EAGLE-3 破 scaling → MTP 联合训练零对齐 → DFlash 并行解耦延迟 → DSpark 半自回归 + 负载调度。[文档·02]

    3 · 3 分钟默背稿（可直接背）¶

    0:00–0:30 · 定调 + 账本¶

    Decode 小 batch 是 memory-bound：权重贵、算力闲。投机 = 便宜 draft 猜 k → target 一次前向并行验。
    单 token 延迟 (T_draft+T_verify)/τ；要它小于纯 target 一步才划算。大 batch 变 compute-bound，verify 不再免费。[机制·01B/02]

    0:30–1:30 · Medusa → EAGLE 三代¶

    Medusa：多头挂在 target 顶层，省独立 draft；但头之间不看彼此，长草稿一致性差，typical acceptance 也别跟严格拒绝采样混谈。[文档·02]
    EAGLE-1：在更平滑的特征层做自回归，并把上一步真实采样 token 喂回去，序列依赖比 Medusa 强；头仍小，复用 emb/LM head。[文档·02]
    EAGLE-2：发现草稿置信度与真实接受率高度相关，用置信度动态决定树往哪长——难位置少扩、易位置多扩，免重训抬接受长度。[文档·02]
    EAGLE-3：发现特征预测损失成了 scaling 枷锁，改成直接预测 token + 多层特征融合，并在训练时模拟多步自回归；接受率再抬，但 draft 本质还是串行。[文档·02]

    1:30–2:20 · MTP → DFlash → DSpark¶

    MTP：DeepSeek 把顺序 MTP 模块放进预训练；推理时当 draft 头，共享表征所以对齐好——前提是模型自带这些权重，第三方套不上就另训。[文档·02]
    DFlash：嫌串行 draft 的 T_draft∝k，改用 block diffusion 一次并行出整块；靠把 target 多层特征注入 draft KV 保住接受率。代价是块内 token 互相看不见，块越长后缀越烂——suffix decay。[文档·02]
    DSpark：主干仍并行（速度），加一层几乎免费的低秩 Markov 头从左到右偏置校正（连贯）；再用 confidence 按实时 GPU 负载裁送验长度——负载低多验抬 τ，负载高少验防 T_verify 暗涨。三管齐下修的是同一个延迟公式。[文档·02 / 01B]

    2:20–3:00 · MindIE 三插件 + 红线一句¶

    MindIE 产品线不是跟论文一一抄名，而是三插件：MTP（模型自带草稿头前向）、Lookahead（Jacobi 迭代出候选）、Memory Decoding（trie/记忆出候选）。骨架同构——候选来源不同，verify 都走**贪心逐位比对**；与 vLLM 默认概率拒绝采样不是同一套。[文档·02]
    互斥：MTP ↔ 并行解码（LA/Memory）文档互斥；LA ↔ Memory 互斥。MTP×结构化：Serving 入口硬互斥，插件层仍未联调——细则翻 23，勿说可同开。[代码·01B/23]

    4 · MindIE 三插件对照（一句表）¶

    插件 候选从哪来 对照一句
    MTP 模型自带 MTP 层真实小前向 ≈ vLLM MTPSpeculator；DeepSeek 线上主路径之一
    Lookahead Jacobi 迭代并行猜 并行解码族；与 MTP 互斥
    Memory Decoding 历史/trie 记忆出候选 并行解码族；与 LA 互斥

    对照金句：vLLM = drafter 多态 + verifier 统一（拒绝采样）；MindIE = Plugin 多态 + 贪心 verify 统一。[文档·02]

    共享 Plugin 钩子（记名即可，不背实现）：model_inputs_update / sample_preprocess / plugin_verify / plugin_cache_update。[文档·02]

    5 · 易混三刀（开口纠偏）¶

    易混 正确口径
    DFlash = DSpark DFlash 纯并行去噪；DSpark = DFlash 类主干 + Markov + 负载调度
    MindIE 无损 = vLLM 无损 MindIE 贪心下 ≡ 自回归；采样场景收窄后处理；≠ 默认概率拒绝采样
    「演进下一篇一定更快」 无损≠永远加速；大 batch 吞吐常↓；无曲线只谈定性

    提高接受率五手段（点名才展开，母本 01 B）：看 target 内部表征 / 蒸馏对齐 / 联合训练 / 动态树 / 按置信止损或按负载裁 k。[文档·02]

    6 · 快问 10（10–20s / 题）¶

    # 问 答要点
    1 Medusa 解决啥？ 去独立 draft；多头零额外部署。[文档]
    2 EAGLE 相对 Medusa？ 特征层 + 真实 token 反馈；序列依赖更强。[文档]
    3 EAGLE-2 一手？ 置信度动态长树，免重训抬 τ。[文档]
    4 EAGLE-3 破什么？ 去 l_fea；直接预测 token + 多层融合。[文档]
    5 MTP 为何对齐好？ 与主干联合训练，共享表征。[文档]
    6 DFlash 解决啥？ 并行出块，解耦 T_draft 与 k；易 suffix decay。[文档]
    7 DSpark 相对 DFlash？ Markov 补连贯 + 按负载裁 verify。[文档]
    8 MindIE 三插件？ MTP / Lookahead / Memory；候选源不同、贪心 verify。[文档]
    9 MindIE verify ≠ vLLM？ 贪心逐位比对 ≠ 默认 min(1,p/q) 拒绝采样。[文档]
    10 与结构化同开？ 入口硬互斥；插件未打通。见 23。[代码]

    7 · 第三层追问 2 连（演进向；白板向见 32）¶

    ① Markov 头为何几乎不涨 T_draft？
    → 主干一次并行出 base_logits；头是 V×r/r×V 低秩偏置，每步≈embedding + 小 GEMM，不是再跑一层 transformer；可进同一 CUDA Graph。[文档·01B/02]
    → 边界：rank 以模型配置为准，未查权重勿报死数。

    ② 为何还要发明 DSpark（面试官「上周新论文」）？
    → DFlash 修了串行 draft 延迟，但留下块内不连贯 + 高并发下 verify 暗涨；DSpark 用半自回归修 τ、用负载调度兜 T_verify——专修上一代劣势场景。[文档·02]

    8 · 一页抄写版¶

    准：Medusa多头 → EAGLE特征AR → E2动态树 → E3去l_fea → MTP联合训

Medusa: 去独立draft
EAGLE1: 特征层+真实token反馈
EAGLE2: 置信度动态树

MTP:    预训练草稿头，对齐好
DFlash: 并行去噪；suffix decay
DSpark: Markov连贯 + 负载调度verify

MindIE: MTP | LA(Jacobi) | Memory(trie)
        互斥；贪心verify ≠ vLLM拒绝采样
